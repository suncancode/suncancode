# Regression

---

## 1. Introduction & Historical Background

Regression is a **supervised learning** method for prediction tasks (and with modification, classification).

**History:**
- First appeared ~1885, developed by **Francis Galton** studying hereditary stature (height of parents vs. children)
- Galton did not connect regression with the **least squares method** — that link was discovered 80 years later

**Three types of linear regression:**

| Type | Inputs | Outputs |
|---|---|---|
| Simple Linear Regression | 1 | 1 |
| Multiple Regression | many | 1 |
| Multivariate Regression | many | many |

**Core idea:** The output (dependent variable) is assumed to be **linearly related** to the inputs (independent variables). Independent variables can be formed from **nonlinear basis functions** of the raw inputs — the model is still "linear" because it is linear in its **parameters**, not necessarily in x.

> **Goal:** Determine the coefficients of this relationship → use the equation to predict on new unseen inputs.

---

## 2. Theoretical Formulation

### Setup

| Symbol | Meaning |
|---|---|
| 𝒳 | Input space |
| 𝒴 ⊂ ℝ | Output space (real-valued) |
| 𝒟 | Unknown distribution over 𝒳 |
| f : 𝒳 → 𝒴 | True target labelling function |
| S = {(x₁,y₁),...,(xₘ,yₘ)} | Labelled training sample, drawn i.i.d from 𝒟 |
| ℋ | Hypothesis set — functions the learner considers |

### Loss Functions

**Squared loss** (most common):
$$\mathcal{L}(y, \hat{y}) = ||y - \hat{y}||^2$$

**General Lₚ loss:**
$$\mathcal{L}_p(y, \hat{y}) = ||y - \hat{y}||^p \quad p \geq 1$$

### Two Types of Error

**Generalization error** (true risk) — what we *want* to minimize but **cannot compute directly**:
$$R(h) = \mathbb{E}_{x \sim \mathcal{D}}[\mathcal{L}(h(x), f(x))]$$

**Empirical error** (training error) — what we *can* compute:
$$\hat{R}(h) = \frac{1}{m} \sum_{i=1}^{m} \mathcal{L}(h(x_i), y_i)$$

> **Key insight:** We minimize empirical error during training, hoping it approximates the true generalization error.

## 3. Linear Regression

### Model

Let **Φ : 𝒳 → ℝᴺ** be a feature mapping. The hypothesis family is:
$$\mathcal{H} = \{x \mapsto \mathbf{w} \cdot \Phi(x) + b : \mathbf{w} \in \mathbb{R}^N, b \in \mathbb{R}\}$$

> **Note:** Φ(x) can be nonlinear — the model is linear in **w**, not necessarily in x.

### Optimization Problem

Find **w, b** minimizing mean squared error over training set:
$$\min_{w,b} \frac{1}{m} \sum_{i=1}^{m} (\mathbf{w} \cdot \Phi(x_i) + b - y_i)^2 \tag{4}$$

### Matrix Form

Absorb bias b into w (append 1 to each feature vector), then write:

$$\mathbf{X} = \begin{bmatrix} \Phi(x_1) & \cdots & \Phi(x_m) \\ 1 & \cdots & 1 \end{bmatrix}, \quad \mathbf{W} = \begin{bmatrix} w_1 \\ \vdots \\ w_N \\ 1 \end{bmatrix}, \quad \mathbf{Y} = \begin{bmatrix} y_1 \\ \vdots \\ y_m \end{bmatrix}$$

Compact form:
$$\min_{\mathbf{W}} F(\mathbf{W}) = \frac{1}{m}||\mathbf{X}^T\mathbf{W} - \mathbf{Y}||^2 \tag{5}$$

**Dimensions check:**
- Xᵀ ∈ ℝ^{m×(N+1)}
- W ∈ ℝ^{N+1}
- XᵀW ∈ ℝᵐ
- Y ∈ ℝᵐ

### Closed-Form Solution (Least Squares)

F(W) is **convex and differentiable** → global minimum found by setting gradient to zero:

$$\nabla F(\mathbf{W}) = 0 \implies \frac{2}{m}\mathbf{X}(\mathbf{X}^T\mathbf{W} - \mathbf{Y}) = 0 \implies \mathbf{X}\mathbf{X}^T\mathbf{W} = \mathbf{X}\mathbf{Y}$$

$$\mathbf{W}^* = \begin{cases} (\mathbf{X}\mathbf{X}^T)^{-1}\mathbf{X}\mathbf{Y} & \text{if } \mathbf{X}\mathbf{X}^T \text{ is invertible} \\ (\mathbf{X}\mathbf{X}^T)^{\dagger}\mathbf{X}\mathbf{Y} & \text{otherwise (pseudo-inverse)} \end{cases} \tag{6}$$

**Geometric interpretation (Figure 2):** The predicted ŷ vector is the **orthogonal projection** of y onto the hyperplane spanned by the feature vectors x₁, x₂.

### Worked Example: "Does money make people happier?" *(Géron 2019)*

**Dataset:** OECD Better Life Index + IMF GDP per capita

| Country | GDP per capita (USD) | Life Satisfaction |
|---|---|---|
| Hungary | 12,240 | 4.9 |
| Korea | 27,195 | 5.8 |
| France | 37,675 | 6.5 |
| Australia | 50,962 | 7.3 |
| United States | 55,805 | 7.2 |

**Step 1 — Model selection:** Plot shows a roughly linear trend → choose linear model:
$$\text{life\_satisfaction} = \theta_0 + \theta_1 \times \text{GDP\_per\_capita} \tag{7}$$

**Step 2 — Different parameters give different lines** (Figure 4):
- θ₀=8, θ₁=−5×10⁻⁵ → downward slope (wrong direction)
- θ₀=4, θ₁=5×10⁻⁵ → upward but too steep
- θ₀=0, θ₁=2×10⁻⁵ → starts too low

**Step 3 — Optimal fit** (Figure 5):
$$\text{life\_satisfaction} = 4.85 + 4.91 \times 10^{-5} \times \text{GDP\_per\_capita} \tag{8}$$

**Step 4 — Predict for Cyprus** (GDP = 22,587, no satisfaction data available):
$$\hat{y} = 4.85 + 4.91 \times 10^{-5} \times 22{,}587 = 5.96$$

### Scikit-Learn Implementation

```python
import sklearn.linear_model

model = sklearn.linear_model.LinearRegression()
model.fit(X, y)
model.predict([[22587]])  # outputs [[5.96242338]]
```

### Limitations of Plain Linear Regression

| Issue | Explanation |
|---|---|
| **High variance** | Low bias but large variance → prone to overfitting |
| **No feature selection** | Cannot identify which variables matter most |
| **No regularization** | Does not control ‖W‖ → no generalization guarantee |
| **Overfit/underfit risk** | Possible on both ends without tuning |

→ These limitations motivate Ridge, Lasso, and Elastic Net.

## 4. Kernel Ridge Regression

### Motivation
Add an **L2 regularization** term to penalize large weights and control variance.

### Optimization Problem

$$\min_{\mathbf{W}} F(\mathbf{W}) = \lambda||\mathbf{W}||^2 + ||\mathbf{X}^T\mathbf{W} - \mathbf{Y}||^2 \tag{9}$$

- **λ** = regularization parameter controlling the trade-off between fitting data and keeping weights small
- **λ‖W‖²** = regularization term (penalizes complexity)
- **‖XᵀW − Y‖²** = empirical MSE (penalizes poor fit)

### Closed-Form Solution

$$\nabla F(\mathbf{W}) = 0 \implies (\mathbf{X}\mathbf{X}^T + \lambda\mathbf{I})\mathbf{W} = \mathbf{X}\mathbf{Y}$$

$$\mathbf{W}^* = (\mathbf{X}\mathbf{X}^T + \lambda\mathbf{I})^{-1}\mathbf{X}\mathbf{Y} \tag{10}$$

> **Why always invertible?** (XXᵀ + λI) has eigenvalues = (non-negative eigenvalues of XXᵀ) + λ > 0 → always positive definite → always invertible. This fixes the potential singularity problem in plain linear regression.

### Alternative (Constrained) Formulation

$$\min_{\mathbf{w}} \sum_{i=1}^{m}(\mathbf{w} \cdot \Phi(x_i) - y_i)^2 \quad \text{subject to} \quad ||\mathbf{w}||^2 \leq \Lambda^2 \tag{11}$$

These two formulations are **equivalent** — each value of λ corresponds to a value of Λ.

### Properties of Ridge Regression

| Property | Details |
|---|---|
| **Model selection** | λ helps weight variables appropriately |
| **Bias-variance control** | λ↑ → bias↑, variance↓ ; λ↓ → bias↓, variance↑ |
| **Choosing λ** | Use **cross-validation** |
| **Shrinkage estimator** | Shrinks least squares weights **toward zero** (but never exactly zero) |
| **Kernel extension** | Works with PDS (positive definite symmetric) kernels → extends to nonlinear regression |

### Scikit-Learn Implementation

```python
# Closed-form (Cholesky factorization)
from sklearn.linear_model import Ridge
ridge_reg = Ridge(alpha=1, solver="cholesky")
ridge_reg.fit(X, y)
ridge_reg.predict([[1.5]])  # array([[1.55071465]])

# Stochastic Gradient Descent (for large datasets / neural networks)
sgd_reg = SGDRegressor(penalty="l2")
sgd_reg.fit(X, y.ravel())
sgd_reg.predict([[1.5]])    # array([1.47012588])
```

> **When to use SGD:** Large number of data points and features. This is the standard approach for training neural networks.

## 5. Lasso Regression

### Motivation: Variable Selection

Beyond regularization, we often want to know **which features actually matter**. Two classical approaches:

- **Backward elimination:** Start with all features, iteratively drop the one with smallest F-ratio
- **Forward selection:** Start with no features, iteratively add the one with largest F-ratio

$$F = \frac{(RSS_0 - RSS_1)/(df_0 - df_1)}{RSS_1/df_1} \tag{12}$$

Lasso achieves variable selection **automatically and simultaneously** with fitting.

### What is Lasso?

**Lasso = Least Absolute Shrinkage and Selection Operator**

Combines **variable subset selection** and **shrinkage** in one step.

### Optimization Problem

Uses **L1 norm** instead of L2:

$$\min_{\mathbf{w},b} F(\mathbf{w},b) = \lambda||\mathbf{w}||_1 + \sum_{i=1}^{m}(\mathbf{w} \cdot x_i + b - y_i)^2 \tag{14}$$

Equivalently (constrained form):
$$\min_{\mathbf{w},b} \sum_{i=1}^{m}(\mathbf{w} \cdot x_i + b - y_i)^2 \quad \text{subject to} \quad ||\mathbf{w}||_1 \leq \Lambda_1$$

This is a **Quadratic Program (QP)** — solvable by QP solvers.

### Key Property: Sparse Solution

> **Lasso produces sparse w** — many weights become **exactly zero**, effectively removing those features from the model.

**Why does L1 produce sparsity? (Figure 6)**

The geometry explains it:
```
L1 ball = diamond shape  (has sharp corners)
L2 ball = circle shape   (smooth, no corners)

Optimal solution = intersection of loss ellipsoid with constraint ball

L1: intersection tends to land at a CORNER → some coordinates = 0 (sparse)
L2: intersection lands on a smooth curve → coordinates near 0 but not exactly
```

| | Ridge (L2) | Lasso (L1) |
|---|---|---|
| Constraint shape | Circle (smooth) | Diamond (corners) |
| Weight behavior | Shrinks toward 0, rarely = 0 | Pushes some weights to exactly 0 |
| Feature selection | No | **Yes — automatic** |
| Kernel extension | Easy (PDS kernels) | Not straightforward |

### Scikit-Learn Implementation

```python
from sklearn.linear_model import Lasso
lasso_reg = Lasso(alpha=0.1)
lasso_reg.fit(X, y)
lasso_reg.predict([[1.5]])  # array([1.53788174])
```

## 6. Elastic Net Regression

### Motivation

Lasso can be unstable when features are **highly correlated** — it arbitrarily selects one and drops the rest. Elastic Net combines both penalties to handle this.

### Optimization Problem

$$\min_{\mathbf{w},b} F(\mathbf{w},b) = \lambda||\mathbf{w}||_1 + \beta||\mathbf{W}||_2^2 + \sum_{i=1}^{m}(\mathbf{w} \cdot x_i + b - y_i)^2 \tag{15}$$

- **λ‖w‖₁** → sparsity (from Lasso)
- **β‖W‖₂²** → stability with correlated features (from Ridge)

### Two-Stage Estimation

1. Find **Ridge regression coefficients**
2. Apply **Lasso-style shrinkage** to those coefficients

### Penalty Norm Geometry (Figure 7)

```
Lasso ball  = diamond (sharp corners)
Ridge ball  = circle  (smooth)
Elastic Net = rounded diamond (between the two)
```

### Partitioned Feature Space

When features are naturally grouped, Elastic Net can select or omit **entire groups** using the **L₂,₁ mixed norm** (combination of L1 and L2).

### Scikit-Learn Implementation

```python
from sklearn.linear_model import ElasticNet
elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)
elastic_net.fit(X, y)
elastic_net.predict([[1.5]])  # array([1.54333232])
```

`l1_ratio=0.5` means equal weight to L1 and L2 penalties.

## 7. Bias-Variance Trade-off

**Prediction error decomposes into three parts:**

$$\text{Prediction Error} = \underbrace{\sigma^2}_{\text{irreducible}} + \underbrace{\text{Bias}^2}_{\text{underfitting}} + \underbrace{\text{Variance}}_{\text{overfitting}}$$

| Component | Definition | Controllable? |
|---|---|---|
| **Irreducible error** | Variance of the test target itself (noise in data) | ❌ No |
| **Bias²** | Squared difference between true mean and expected estimate | ✅ Yes |
| **Variance** | How much the estimate fluctuates across different training sets | ✅ Yes |

**Figure 8 — Training vs. Test Error:**
```
Prediction
Error
  │\
  │ \  ← Test error (U-shaped)
  │  \/‾‾‾‾‾‾‾
  │   \  ← optimal point
  │    \___________  ← Training error (monotone decrease)
  └──────────────────────── Model Complexity
     Low                 High
  High Bias           High Variance
  Low Variance         Low Bias
```

**How regularization fits in:**

| Method | Effect on Bias | Effect on Variance |
|---|---|---|
| λ↑ (more regularization) | Bias ↑ | Variance ↓ |
| λ↓ (less regularization) | Bias ↓ | Variance ↑ |
| λ = 0 (no regularization) | Lowest Bias | Highest Variance (plain linear reg.) |

→ Choose optimal λ using **Cross-Validation**.

## 8. Dealing with Data

### Non-representative Data

> Training data must share the **same underlying probability distribution** as production data — otherwise the model will generalize poorly.

**Two causes and solutions:**

| Problem | Cause | Solution |
|---|---|---|
| **Sampling noise** | Too few samples → random fluctuations | Collect more data |
| **Sampling bias** | Sample not representative of population | Ensure representative sampling strategy |

### Poor Quality Data

| Issue | Solution |
|---|---|
| **Errors / noise** | Clean manually or use robust algorithms |
| **Outliers** | Discard or fix manually |
| **Missing values (many)** | Drop the entire feature |
| **Missing values (few)** | Replace with median or mean |
| **Missing values (uncertain)** | Train two models: one with, one without the feature |

### Irrelevant Features

Features that add no useful information hurt model performance and increase variance.

**Three approaches:**

| Approach | Description | Example |
|---|---|---|
| **Feature selection** | Select most useful features from existing set | Lasso, Elastic Net (automatic); backward/forward selection |
| **Feature extraction** | Combine features to create more informative ones | PCA (dimensionality reduction) |
| **Domain knowledge** | Manually remove known-irrelevant features | — |

> **Further reading:** Chapter 2 of Géron (2019) — end-to-end ML system design.

## 9. Summary & Comparison

### The Four Regression Methods

| Method | Loss Function | Regularization | Key Property | Use When |
|---|---|---|---|---|
| **Linear** | ‖XᵀW−Y‖² | None | Simple, closed-form | Small data, few features |
| **Ridge** | λ‖W‖² + ‖XᵀW−Y‖² | L2 | Shrinks w → 0, always invertible | Many features, all matter |
| **Lasso** | λ‖w‖₁ + ‖Xᵀw−y‖² | L1 | Exact zeros → feature selection | Many features, few important |
| **Elastic Net** | λ‖w‖₁+β‖W‖₂²+‖Xᵀw−y‖² | L1+L2 | Stable with correlated features | Correlated / grouped features |

### Solution Methods

| Scenario | Solver |
|---|---|
| Small/medium data, closed-form exists | Normal equations / Cholesky factorization |
| Large data / neural networks | Stochastic Gradient Descent (SGD) |
| Ridge not invertible | Pseudo-inverse |

### Key Equations Reference

```
Linear Regression:    W* = (XXᵀ)⁻¹ XY

Ridge Regression:     W* = (XXᵀ + λI)⁻¹ XY
                      → λI ensures always invertible

Lasso Regression:     No closed form → QP solver or coordinate descent
                      min λ‖w‖₁ + Σ(w·xᵢ + b - yᵢ)²

Elastic Net:          min λ‖w‖₁ + β‖W‖₂² + Σ(w·xᵢ + b - yᵢ)²
                      Two-stage: Ridge coefficients → Lasso shrinkage
```

### The Big Picture

```
Problem: predict continuous output y from features x
          ↓
Choose model: ŷ = wᵀΦ(x) + b
          ↓
Choose regularization based on your situation:
  No concern about overfit    → Linear Regression
  All features matter          → Ridge  (L2)
  Need feature selection       → Lasso  (L1)
  Correlated features          → Elastic Net (L1+L2)
          ↓
Find optimal λ via Cross-Validation
          ↓
Evaluate on held-out test set (generalization error)
```
