## 1. Big Picture — Why Neural Networks?

Before anything else: why did we need neural networks if we already had regression and classification?

**Regression and classification** work well when the relationship between input and output is **linear or near-linear**. But real-world problems — recognising faces, translating languages, predicting weather — are deeply **non-linear** and **high-dimensional**. A flat decision boundary simply cannot capture this complexity.

Neural Networks were inspired by the biological brain and are designed to model the way the brain performs tasks. Formally (Haykin 2009), a neural network is:

> *"A massively parallel distributed processor made up of simple processing units that has a natural propensity for storing experiential knowledge and making it available for use."*

It resembles the brain in two ways:
- **Knowledge is acquired** through a learning process from the environment.
- **Knowledge is stored** in inter-neuron connection strengths called **synaptic weights**.

The three sessions follow a natural progression:

```
MLP (Session I)  -->  CNN (Session II)  -->  RNN (Session III)
 [tabular data]       [grid/image data]       [sequence/time data]
```

Each architecture is a **specialised inductive bias** — it bakes domain knowledge about data structure directly into the network design.

## Session I — Foundations of Neural Networks

### 2.1 Model of a Neuron

A single artificial neuron is the atomic building block of all neural networks. It has three components:

#### Components

**1. Synapses (weighted connections)**
Each input signal $x_j$ arrives through a connection with weight $\omega_{kj}$. The weight encodes the **importance** of that input. A large positive weight amplifies; a negative weight inhibits.

**2. Adder (linear combiner)**
Sums all weighted inputs:
$$u_k = \sum_{j=1}^{m} \omega_{kj} x_j$$

**3. Activation Function**
Applies a (typically non-linear) transformation to produce the output:
$$y_k = \varphi(u_k + b_k)$$

where $b_k$ is the **bias** term.

#### Role of the Bias

Without bias, the decision boundary is forced to pass through the origin — severely limiting the model. Adding bias $b_k$ shifts the boundary arbitrarily:

$$v_k = u_k + b_k$$

Computationally, bias is absorbed by adding a fixed input $x_0 = +1$ with weight $\omega_{k0} = b_k$, giving a unified formulation:

$$v_k = \sum_{j=0}^{m} \omega_{kj} x_j, \qquad y_k = \varphi(v_k)$$

#### Worked Example

Given: weights $\omega_{10}=0.5,\ \omega_{11}=0.4,\ \omega_{12}=0.6,\ \omega_{13}=0.2$; inputs $x_0=1,\ x_1=1.2,\ x_2=2.0,\ x_3=1.8$; logistic sigmoid activation with $a=0.2$.

**Step 1 — Weighted sum:**
$$v_1 = 1(0.5) + 0.4(1.2) + 0.6(2.0) + 0.2(1.8) = 0.5 + 0.48 + 1.2 + 0.36 = 2.54$$

**Step 2 — Activation:**
$$y_1 = \frac{1}{1 + e^{-0.2 \times 2.54}} = \frac{1}{1 + e^{-0.508}} \approx 0.624$$

### 2.2 Activation Functions

#### Why Do We Need Non-Linear Activation?

This is the most important conceptual point. Without a non-linear activation function, stacking $N$ layers is mathematically equivalent to a single layer:

$$\text{output} = W_N \cdots W_2 W_1 \mathbf{x} = W^* \mathbf{x}$$

The entire deep network collapses to a single linear transformation — identical in expressive power to linear regression. Non-linear activations **break this collapse**, allowing each layer to learn genuinely new representations.

#### Common Activation Functions

| Name | Formula | Range | Differentiable | Common Use |
|------|---------|-------|----------------|------------|
| **Threshold (Step)** | $\varphi(v) = \mathbb{1}[v \geq 0]$ | {0,1} | No (at 0) | Perceptron |
| **Logistic (Sigmoid)** | $\varphi(v) = \frac{1}{1+e^{-av}}$ | (0,1) | Yes | Output (binary) |
| **Tanh** | $\varphi(v) = \tanh(v)$ | (-1,1) | Yes | Hidden layers |
| **ReLU** | $\varphi(v) = \max(0, v)$ | [0, inf) | Yes (except 0) | Hidden layers (default) |
| **Softmax** | $\varphi(v_k) = \frac{e^{v_k}}{\sum_j e^{v_j}}$ | (0,1), sums to 1 | Yes | Output (multi-class) |

**Sigmoid derivative** (important for backprop):
$$\varphi'(v) = \varphi(v)(1 - \varphi(v))$$

**ReLU derivative:**
$$\varphi'(v) = \begin{cases} 1 & v > 0 \\ 0 & v < 0 \end{cases}$$

#### Key Properties of a Good Activation Function

1. **Non-linear** — otherwise the entire network is equivalent to linear regression.
2. **Differentiable** — required for gradient-based learning (backpropagation).
3. **Bounded output for bounded input** — prevents values from exploding.
4. Works with the right **cost function** to make training possible.

#### Sigmoid vs ReLU — A Critical Trade-off

| | Sigmoid | ReLU |
|--|---------|------|
| Vanishing gradient | **Yes** — saturates at extremes, gradient approaches 0 | **No** — gradient = 1 for positive inputs |
| Computation | Expensive (exponential) | Cheap (comparison) |
| Output range | (0, 1) — interpretable as probability | [0, inf) |
| Dying neurons | No | Yes — "Dying ReLU" if neuron always outputs 0 |

ReLU (Nair & Hinton, 2010) is the default choice for hidden layers in modern deep learning. Its key advantage is avoiding vanishing gradients, which was the main barrier to training deep networks.

### 2.3 Network Architectures

#### Single-Layer Feedforward
Input layer connects directly to output layer. Equivalent to logistic regression. Can only solve **linearly separable** problems.

#### Multilayer Feedforward (MLP)
```
Input layer --> Hidden layer 1 --> Hidden layer 2 --> ... --> Output layer
```
- Each hidden layer learns increasingly abstract representations.
- Adding hidden layers allows the network to extract **higher-order statistics** from input data.
- A network is **fully connected** if every neuron in each layer connects to every neuron in the adjacent forward layer.
- **Universal Approximation Theorem**: A single hidden layer MLP with a non-linear activation function can approximate any continuous function to arbitrary precision, given enough neurons.

#### Recurrent Networks
- Unlike feedforward, recurrent networks introduce **feedback loops** — output can return as input.
- Feedback loops + non-linear activations allow modelling **non-linear dynamic systems**.
- *(Covered in depth in Session III)*

### 2.4 Learning Process & Gradient Descent

#### Types of Learning

| Type | How it works | Example |
|------|-------------|---------|
| **Supervised** | Given labelled (x, t) pairs; learn f: x -> t | Image classification |
| **Unsupervised** | No labels; find internal structure of data | Clustering |
| **Reinforcement** | Agent receives rewards/penalties; maximise cumulative reward | Game playing |

#### Supervised Learning as Optimisation

The goal: find weights $\omega$ such that:
$$y = f(\omega, \mathbf{x})$$
minimises the **cost function** (loss). For regression/classification:
$$J(\omega, b) = -\mathbb{E}[\log p_{\text{model}}(y|\mathbf{x})]$$

This is the **cross-entropy** between the training data distribution and the model's predicted distribution.

#### Gradient Descent — From First Principles

**Intuition**: Imagine standing on a hilly landscape in fog, trying to reach the lowest valley. You cannot see the whole landscape, but you can feel the slope under your feet. The optimal strategy: always step in the direction of steepest descent.

**For a scalar function** $f(x)$, the derivative $f'(x)$ tells us the slope at $x$:
- $f'(x) > 0$: function increasing — move **left** to reduce $f$
- $f'(x) < 0$: function decreasing — move **right** to reduce $f$

**Update rule** (from Taylor expansion: $f(x+\epsilon) \approx f(x) + \epsilon f'(x)$):
$$x^{\text{new}} = x - \epsilon f'(x)$$
where $\epsilon > 0$ is the **learning rate** (step size).

**Numerical example** with $f(x) = \frac{1}{2}x^2$, $f'(x) = x$, $\epsilon = 0.1$:

| Starting point | f'(x) | x_new | f(x_new) | Result |
|---|---|---|---|---|
| x = -1 | -1 | -1 - 0.1(-1) = -0.9 | 0.405 < 0.5 | Moved right toward 0 |
| x = 1.5 | 1.5 | 1.5 - 0.1(1.5) = 1.35 | 0.911 < 1.125 | Moved left toward 0 |

**For vector inputs**, the derivative generalises to the **gradient**:
$$\nabla f(\mathbf{x}) = \left[\frac{\partial f}{\partial x_1},\ \frac{\partial f}{\partial x_2},\ \ldots,\ \frac{\partial f}{\partial x_m}\right]^T$$

The gradient points in the direction of **steepest ascent**. To minimise, move opposite to it:
$$\mathbf{x}' = \mathbf{x} - \epsilon \nabla f(\mathbf{x})$$

### 2.5 Perceptron

The perceptron is the simplest neural network: a single neuron with a threshold activation function used for **binary classification**.

**Decision rule:**
- $y = +1$ — classify to class $C_1$
- $y = -1$ — classify to class $C_2$

The perceptron finds a **hyperplane** that separates the two classes:
$$\sum_{i=1}^{m} \omega_i x_i + b = 0$$

Weights are updated using the **error-correction rule** each time a sample is misclassified.

**Critical limitation**: The perceptron can only solve **linearly separable** problems. The classic failure is XOR — two interleaved classes that cannot be separated by a single line. This motivated the Multilayer Perceptron.

---

### 2.6 Multilayer Perceptron & Backpropagation

#### Features of MLP
- Each neuron has a **non-linear, differentiable** activation function.
- One or more **hidden layers** between input and output.
- High connectivity determined by synaptic weights.

#### Training: Backpropagation Algorithm

Training uses **error-correction learning** in two alternating phases:

**Forward Pass** — weights are fixed; input propagates layer by layer to produce output:
$$v_j(n) = \sum_{i=0}^{m} \omega_{ji}(n)\, y_i(n), \qquad y_j(n) = \varphi_j(v_j(n))$$

**Backward Pass** — compare output to desired response, propagate error signal backward, update weights.

#### Error Signal and Cost

Error at output neuron $j$:
$$e_j(n) = d_j(n) - y_j(n)$$

Instantaneous error energy:
$$E_j(n) = \frac{1}{2} e_j^2(n)$$

Total error (summed over all output neurons $C$):
$$E(n) = \sum_{j \in C} E_j(n) = \frac{1}{2} \sum_{j \in C} e_j^2(n)$$

> The factor 1/2 is included for convenience — it cancels when differentiating $e_j^2$, leaving a clean gradient.

#### Deriving the Weight Update via Chain Rule

The correction $\Delta\omega_{ji}$ is proportional to $-\frac{\partial E}{\partial \omega_{ji}}$. Using the chain rule:

$$\frac{\partial E}{\partial \omega_{ji}} = \underbrace{\frac{\partial E}{\partial e_j}}_{e_j(n)} \cdot \underbrace{\frac{\partial e_j}{\partial y_j}}_{-1} \cdot \underbrace{\frac{\partial y_j}{\partial v_j}}_{\varphi'_j(v_j)} \cdot \underbrace{\frac{\partial v_j}{\partial \omega_{ji}}}_{y_i(n)}$$

Combining:
$$\frac{\partial E}{\partial \omega_{ji}} = -e_j(n)\, \varphi'_j(v_j(n))\, y_i(n)$$

**Delta rule** — the weight correction:
$$\Delta\omega_{ji}(n) = \eta\, \underbrace{e_j(n)\, \varphi'_j(v_j(n))}_{\delta_j(n)}\, y_i(n) = \eta\, \delta_j(n)\, y_i(n)$$

where $\delta_j(n)$ is the **local gradient** and $\eta$ is the **learning rate**.

#### Weight Update Rules

**Output neurons** (we know $d_j(n)$ directly):
$$\omega_{ji}^{\text{new}} = \omega_{ji}^{\text{old}} + \eta\, e_j(n)\, \varphi'_j(v_j(n))\, y_i(n)$$

**Hidden neurons** (no direct desired output — error is *back-propagated* from the layer to the right):
$$\delta_j(n) = \varphi'_j(v_j(n)) \sum_k \delta_k(n)\, \omega_{kj}(n)$$
$$\omega_{ji}^{\text{new}} = \omega_{ji}^{\text{old}} + \eta\, \varphi'_j(v_j(n)) \sum_k \delta_k(n)\, \omega_{kj}(n)\, y_i(n)$$

where the sum is over all neurons $k$ in the layer immediately to the right of neuron $j$.

#### The Credit Assignment Problem

Training a hidden neuron is difficult because we do not know what its "correct" output should be. Backpropagation solves this **credit assignment problem** by using the weighted sum of the local gradients of all downstream neurons. Each hidden neuron receives credit or blame proportional to how much it contributed to the final error, mediated by the connecting weights.

#### Why Differentiable Activation Functions Are Required

The hidden-neuron weight update contains $\varphi'_j(v_j(n))$ — the derivative of the activation function. If the activation is not differentiable (like the threshold step function), this term is undefined or zero almost everywhere, and backpropagation cannot work.

### 2.7 Optimizers — Extensions of Gradient Descent

Basic gradient descent can converge slowly or get stuck. The following optimizers improve it:

#### Stochastic Gradient Descent (SGD)
$$g_t \leftarrow \nabla f(\omega_{t-1}), \qquad \omega_t \leftarrow \omega_{t-1} - \eta g_t$$
Updates after each individual sample rather than the full dataset.

#### Classical Momentum
Accumulates a velocity vector to accelerate learning in consistent directions and dampen oscillations:
$$m_t \leftarrow \mu m_{t-1} + g_t, \qquad \omega_t \leftarrow \omega_{t-1} - \eta m_t$$

#### Nesterov's Accelerated Gradient (NAG)
Computes the gradient *after* applying momentum — a "look-ahead" correction that is empirically superior to classical momentum:
$$g_t \leftarrow \nabla f(\omega_{t-1} - \eta\mu m_{t-1}), \qquad m_t \leftarrow \mu m_{t-1} + g_t, \qquad \omega_t \leftarrow \omega_{t-1} - \eta m_t$$

#### AdaGrad
Adapts the learning rate per-parameter — faster for rarely-updated weights, slower for frequently-updated ones:
$$n_t \leftarrow n_{t-1} + g_t^2, \qquad \omega_t \leftarrow \omega_{t-1} - \eta \frac{g_t}{\sqrt{n_t + \epsilon}}$$
**Problem**: accumulated $n_t$ grows forever, eventually halting learning.

#### RMSProp
Fixes AdaGrad's growing gradient by using an exponentially weighted moving average:
$$n_t \leftarrow \nu n_{t-1} + (1-\nu)g_t^2, \qquad \omega_t \leftarrow \omega_{t-1} - \eta \frac{g_t}{\sqrt{n_t + \epsilon}}$$

#### Adam (Adaptive Moment Estimation)
Combines momentum (first moment) and RMSProp (second moment), with bias correction:
$$m_t \leftarrow \mu m_{t-1} + (1-\mu)g_t$$
$$n_t \leftarrow \nu n_{t-1} + (1-\nu)g_t^2$$
$$\hat{m}_t = \frac{m_t}{1-\mu^t}, \qquad \hat{n}_t = \frac{n_t}{1-\nu^t}$$
$$\omega_t \leftarrow \omega_{t-1} - \eta \frac{\hat{m}_t}{\sqrt{\hat{n}_t} + \epsilon}$$

Adam is the most widely used optimizer in practice due to its robustness and fast convergence.

| Optimizer | Adapts LR? | Uses Momentum? | Notes |
|-----------|-----------|----------------|-------|
| SGD | No | No | Simple baseline |
| Momentum | No | Yes | Faster on consistent gradients |
| NAG | No | Yes (look-ahead) | Better than classical momentum |
| AdaGrad | Yes | No | Good for sparse data; LR decays to zero |
| RMSProp | Yes | No | Fixes AdaGrad; good for RNNs |
| **Adam** | **Yes** | **Yes** | **Default choice; most robust** |

## Session II — Convolutional Neural Networks (CNN)

### 3.1 Why CNN? The Problem with MLP on Images

Consider a modest 224x224 RGB image — that is 224 x 224 x 3 = 150,528 input values. A fully connected hidden layer with just 1,000 neurons would require **150 million parameters** for the first layer alone. This leads to:

1. **Computational infeasibility** — too many parameters to train.
2. **Overfitting** — the model memorises training data rather than generalising.
3. **No spatial awareness** — a fully connected layer treats pixel (0,0) and pixel (223,223) as completely independent, ignoring the fact that **nearby pixels are strongly correlated**.

CNN solves all three by building **prior knowledge about image structure** directly into the architecture through two key constraints:
- **Local connections** (receptive fields): each neuron only looks at a small spatial region.
- **Weight sharing**: the same filter (weights) is applied across all spatial positions.

**Contrast with MLP:**
```
MLP:  v_i = W * v_{i-1}              (dense matrix multiplication)
CNN:  v_j = sum_i(w_i * x_{i+j-1})  (convolution — same weights w reused across positions)
```

---

### 3.2 The Convolution Operation

#### Mathematical Definition

Given input $x(n)$ and kernel $\omega(n)$, their **convolution** is:
$$s(n) = \sum_k x(k)\, \omega(n-k)$$

Interpretation: flip the kernel, slide it across the input, and at each position compute the dot product of overlapping values.

**Terminology in neural networks:**
- $x(n)$ = input (e.g., image)
- $\omega(n)$ = **kernel** (or filter) — the learnable weights
- $s(n)$ = **feature map** (output of the convolution)

#### 2D Convolution (for images)

$$s(n,m) = \sum_{k,l} x(k,l)\, \omega(n-k,\, m-l)$$

In practice, ML frameworks implement **cross-correlation** (without flipping the kernel) and call it "convolution":
$$s(n,m) = \sum_{k,l} x(k,l)\, \omega(n+k,\, m+l)$$

The distinction does not matter for learning because the kernel is learned anyway — whether flipped or not, the network finds the right values.

#### Concrete 2D Example

Input (4x3) convolved with Kernel (2x2):
```
Input:              Kernel:
| a  b  c  d |      | w  x |
| e  f  g  h |      | y  z |
| i  j  k  l |

Output (3x2):
| aw+bx+ey+fz   bw+cx+fy+gz   cw+dx+gy+hz |
| ew+fx+iy+jz   fw+gx+jy+kz   gw+hx+ky+lz |
```

Each output cell is a dot product between the kernel and a local patch of the input. As the kernel slides across the input, it detects the same **local pattern** at every position.

#### Key Parameters

- **Kernel size**: spatial extent of the filter (e.g., 3x3, 5x5). Larger kernels see more context but have more parameters.
- **Stride**: how many pixels the kernel moves at each step. Stride 1 gives dense output; Stride 2 halves output in each dimension.
- **Padding**:
  - `VALID` — no padding; output is smaller than input.
  - `SAME` — zero-pad input so output has the same spatial size as input.

---

### 3.3 Three Core Benefits of Convolution

#### 1. Sparse Interactions (Local Connectivity)
In a fully connected layer, every output depends on every input. In a conv layer, each output depends only on a **small local region** (the receptive field). This reduces parameters and forces the network to learn **local features** (edges, textures, corners) that are meaningful in images.

As you go deeper, each neuron's **effective receptive field** grows — it indirectly "sees" larger portions of the input, allowing deep layers to detect complex global patterns while early layers detect simple local ones.

#### 2. Parameter Sharing (Weight Sharing)
The same kernel is applied at **every spatial position**. This means:
- Instead of learning one set of weights per location, we learn **one set** shared everywhere.
- A filter that detects horizontal edges at position (10,10) is automatically applied at (50,50) — the network does not need to relearn the same feature at every location.
- Dramatic reduction in parameters: a 3x3 filter has only 9 weights regardless of image size.

#### 3. Equivariant Representation (Translation Equivariance)
Convolution is **equivariant to translation**: if the input shifts, the feature map shifts by the same amount.

Formally, if $g$ shifts image $I$ by one pixel — $I'(x,y) = I(x-1, y)$ — then:
$$\text{Conv}(g(I)) = g(\text{Conv}(I))$$

Applying the convolution before or after the shift gives the same result. This is the spatial analogue of **linear time invariance** in signal processing.

> **Equivariance vs Invariance**: Equivariance means the output *moves with* the input transformation. Invariance (achieved by pooling) means the output *does not change* despite the input transformation.

---

### 3.4 Pooling

A pooling layer **subsamples** the feature map by replacing a neighbourhood of values with a single summary statistic.

#### Types of Pooling

**Max Pooling** — takes the maximum value in each window:
```
Input region: [0.1, 1.0, 0.2, 0.1]   -->   Max Pool output: 1.0
```

**Average Pooling** — takes the mean of values in each window.

**L2-norm Pooling** — takes the L2 norm of values in the window.

#### Why Pooling?

1. **Spatial dimension reduction** — smaller feature maps give fewer parameters in subsequent layers.
2. **Translation invariance** — if the feature shifts slightly within the pooling window, the output remains the same, making representations robust to small distortions.
3. **Reduces overfitting** — fewer parameters to overfit.

**Illustration of translation invariance** (max pooling, width 3, stride 1):
```
Original input:  [0.1,  1.0,  0.2,  0.1]
Max pool output: [1.0,  1.0,  0.2]

Shifted input:   [0.3,  0.1,  1.0,  0.2]
Max pool output: [1.0,  1.0,  1.0]   <-- same dominant activation preserved
```

#### Pooling Parameters
- **Pool size**: the spatial extent of the pooling window.
- **Stride**: how far the window moves between applications. Stride > 1 causes **downsampling**.

---

### 3.5 Typical CNN Architecture

A standard CNN alternates between convolution/activation and pooling, then flattens into fully connected layers:

```
Input Image
    |
[Conv --> ReLU] x n           <- feature extraction
    |
[Pooling]                     <- spatial reduction + invariance
    |
[Conv --> ReLU] x n
    |
[Pooling]
    |
   ...
    |
Flatten
    |
[Fully Connected --> ReLU]    <- classification head
    |
[Fully Connected --> Softmax]
    |
Output (class probabilities)
```

**Example — Hand-written character recognition CNN (Haykin 2009):**

| Layer | Type | Feature Maps | Size | Receptive Field |
|-------|------|-------------|------|----------------|
| Input | — | 1 | 28x28 | — |
| C1 | Convolution | 4 | 24x24 | 5x5 |
| S2 | Subsampling | 4 | 12x12 | 2x2 |
| C3 | Convolution | 12 | 8x8 | — |
| S4 | Subsampling | 12 | 4x4 | — |
| Output | Convolution | 26 | 1x1 | 4x4 |

The 26 output neurons correspond to the 26 possible characters.

### 3.6 Landmark CNN Architectures

#### LeNet-5 (LeCun, 1998) — The Pioneer

The first successful CNN, applied to handwritten digit recognition (MNIST). Introduced the core Conv -> Pool -> Conv -> Pool -> FC pattern.

| Layer | Type | Maps | Size | Kernel | Stride | Activation |
|-------|------|------|------|--------|--------|------------|
| In | Input | 1 | 32x32 | — | — | — |
| C1 | Conv | 6 | 28x28 | 5x5 | 1 | tanh |
| S2 | Avg Pool | 6 | 14x14 | 2x2 | 2 | tanh |
| C3 | Conv | 16 | 10x10 | 5x5 | 1 | tanh |
| S4 | Avg Pool | 16 | 5x5 | 2x2 | 2 | tanh |
| C5 | Conv | 120 | 1x1 | 5x5 | 1 | tanh |
| F6 | Fully Conn | — | 84 | — | — | tanh |
| Out | Fully Conn | — | 10 | — | — | RBF |

Note: MNIST images are 28x28, so padding to 32x32 was required.

---

#### AlexNet (Krizhevsky, Sutskever, Hinton — 2012) — The Deep Learning Revolution

Won the 2012 ImageNet LSVRC competition by a large margin, sparking the modern deep learning era.

| Layer | Type | Maps | Size | Kernel | Stride | Padding | Activation |
|-------|------|------|------|--------|--------|---------|------------|
| In | Input | 3 (RGB) | 224x224 | — | — | — | — |
| C1 | Conv | 96 | 55x55 | 11x11 | 4 | SAME | ReLU |
| S2 | Max Pool | 96 | 27x27 | 3x3 | 2 | VALID | — |
| C3 | Conv | 256 | 27x27 | 5x5 | 1 | SAME | ReLU |
| S4 | Max Pool | 256 | 13x13 | 3x3 | 2 | VALID | — |
| C5 | Conv | 384 | 13x13 | 3x3 | 1 | SAME | ReLU |
| C6 | Conv | 384 | 13x13 | 3x3 | 1 | SAME | ReLU |
| C7 | Conv | 256 | 13x13 | 3x3 | 1 | SAME | ReLU |
| F8 | Fully Conn | — | 4096 | — | — | — | ReLU |
| F9 | Fully Conn | — | 4096 | — | — | — | ReLU |
| Out | Fully Conn | — | 1000 | — | — | — | Softmax |

**Key innovations over LeNet:**
- First widespread use of **ReLU** — enabling faster training and avoiding vanishing gradients.
- **Max pooling** instead of average pooling.
- **Stacked conv layers** (C5, C6, C7) without pooling between them.
- **Dropout** in layers F8 and F9 to prevent overfitting.
- **Data augmentation**: random crops, horizontal flips, colour jitter.
- **Local Response Normalisation (LRN)**: strongly activated neurons inhibit neighbours in adjacent feature maps, encouraging specialisation.

---

#### GoogLeNet / Inception (Szegedy et al. — 2014)

Won the 2014 ImageNet LSVRC with top-5 error < 7%.

**Key innovation — The Inception Module:**
Instead of choosing between 1x1, 3x3, or 5x5 convolutions, apply **all of them in parallel** and concatenate the results along the depth axis:

```
                    Input
                      |
        +-------------+-------------+
    1x1 Conv      3x3 Conv      5x5 Conv    Max Pool 3x3
                 (with 1x1)   (with 1x1)    (with 1x1)
        +-------------+-------------+
                 Depth Concat
                      |
                   Output
```

The `1x1` convolutions before the `3x3` and `5x5` are **bottleneck layers** that reduce the number of channels, dramatically cutting computation.

**Key stats**: approximately 6 million parameters vs AlexNet's 60 million — **10x more efficient**.

---

#### ResNet (He et al., Microsoft Research — 2015)

Won the 2015 ImageNet LSVRC. Introduced networks as deep as 152 layers.

**Motivation — The Degradation Problem:**
Simply adding more layers *hurts* training accuracy. This is not overfitting — deeper networks are harder to optimise with standard backpropagation.

**Solution — Residual Learning / Skip Connections:**

Instead of learning the direct mapping $H(x)$, let the layers learn the **residual**:
$$\mathcal{F}(x) := H(x) - x$$

The actual output is then:
$$H(x) = \mathcal{F}(x) + x$$

The $x$ is added directly via a **skip connection** that bypasses one or more layers.

```
   x ----------------------------------------+
   |                                          |  (skip connection)
[Conv --> BN --> ReLU]                        |
   |                                          |
[Conv --> BN]                                 |
   |                                          |
   + <----------------------------------------+
   |
 ReLU
   |
  H(x) = F(x) + x
```

**Why does this work?** If the optimal transformation for a block is close to the identity, it is easier to learn F(x) ≈ 0 than to learn H(x) ≈ x from scratch. Skip connections provide a "highway" for gradients to flow directly backward through many layers, solving the vanishing gradient problem in very deep networks.

## Session III — Recurrent Neural Networks (RNN)

### 4.1 Why RNN? The Problem with MLP/CNN on Sequences

MLP and CNN process **fixed-size, independent inputs**. They have no concept of time or order. This is fundamentally insufficient for:

| Task | Input | Output |
|------|-------|--------|
| Sentiment analysis | Variable-length text | Single label |
| Machine translation | Sentence in language A | Sentence in language B |
| Speech recognition | Audio sequence | Text sequence |
| Temperature prediction | Historical readings | Future readings |
| Image captioning | Image (2D) | Text sequence |

**Two key challenges with sequence data:**
1. **Variable length**: inputs and outputs can have different lengths across examples.
2. **Temporal dependency**: the meaning of $x(t)$ depends on what came before — order matters.

**Biological inspiration**: Biological intelligence processes information **incrementally**, maintaining an internal model updated continuously as new information arrives. RNNs adopt this principle.

---

### 4.2 The Vanilla RNN

#### Core Idea — The Hidden State

An RNN maintains a **hidden state** $h(t)$ that acts as the network's "memory" of everything it has seen so far. At each time step $t$:

1. New input $x(t)$ arrives.
2. The hidden state is updated using $x(t)$ **and** the previous hidden state $h(t-1)$.
3. An output $y(t)$ is produced from $h(t)$.

**The fundamental equations:**
$$h(t) = \tanh(W_{hh}\, h(t-1) + W_{xh}\, x(t) + b_h)$$
$$y(t) = W_{hy}\, h(t) + b_y$$

where:
- $W_{xh}$: input-to-hidden weight matrix
- $W_{hh}$: hidden-to-hidden (recurrent) weight matrix
- $W_{hy}$: hidden-to-output weight matrix
- $b_h, b_y$: bias vectors

**Critical property**: The **same weight matrices** are used at **every time step**. This is parameter sharing in the temporal dimension, analogous to how CNN kernels are shared spatially.

#### Unrolled RNN

The RNN can be "unrolled" through time to visualise the computation:

```
x(1)    x(2)    x(3)    ...   x(T)
  |       |       |             |
[RNN] -> [RNN] -> [RNN] -> ... -> [RNN]
  |       |       |             |
h(1) -> h(2) -> h(3) ->  ...  -> h(T)
  |       |       |             |
y(1)   y(2)   y(3)           y(T)
```

The horizontal arrows are **recurrent connections** — how information flows through time.

#### Training: Backpropagation Through Time (BPTT)

Backpropagation in RNNs unrolls the network through time and applies the chain rule. However, this suffers from two major problems on long sequences:

- **Vanishing gradients**: gradients shrink exponentially through many time steps — the network cannot learn long-range dependencies.
- **Exploding gradients**: gradients grow exponentially — training diverges. Typically handled by **gradient clipping**.

LSTM and GRU are specifically designed to address vanishing gradients.

---

### 4.3 RNN Computational Graphs & Modes

Different tasks require different input-output configurations:

#### Many-to-Many
Input and output are both sequences of the same length. Use case: video frame-by-frame captioning, part-of-speech tagging.
```
x(1) x(2) x(3) ... x(T)
  |    |    |         |
[RNN]->[RNN]->[RNN]->...[RNN]
  |    |    |         |
y(1) y(2) y(3)      y(T)
```

#### Many-to-One
Sequence input, single output. Use case: sentiment classification.
```
x(1) x(2) x(3) ... x(T)
  |    |    |         |
[RNN]->[RNN]->[RNN]->...[RNN]
                         |
                         y
```

#### One-to-Many
Single input, sequence output. Use case: image captioning (input = image features).

#### Sequence-to-Sequence (Encoder-Decoder)
Used for machine translation. Two RNNs chained together:
```
[Encoder RNN]  -->  context vector  -->  [Decoder RNN]
     |                                         |
  "hello"                                  "bonjour"
  "world"                                      |
 (English)                                  "monde"
                                           (French)
```

The encoder reads the entire input and compresses it into a fixed-size **context vector** (final hidden state). The decoder generates the output sequence from this context vector.

---

### 4.4 LSTM — Long Short-Term Memory

Introduced by Hochreiter & Schmidhuber (1997). LSTM adds a second state — the **cell state** $c(t)$ — and a system of **gates** to control information flow.

#### Two States: Short-term vs Long-term Memory

- **$h(t)$** — hidden state ("short-term memory"): what is relevant right now.
- **$c(t)$** — cell state ("long-term memory"): information that can persist over many time steps.

The cell state acts as a **conveyor belt** running along the sequence. Gates add or remove information from it precisely.

#### The Three Gates

Each gate produces values in (0, 1) via sigmoid activation. Multiplying by 0 **closes** the gate; multiplying by 1 **opens** it.

**1. Forget Gate** $f(t)$ — decides what to erase from long-term memory:
$$f(t) = \sigma(W_{xf}\, x(t) + W_{hf}\, h(t-1) + b_f)$$
Element-wise multiplied with $c(t-1)$ to selectively erase old content.

*Intuition*: When starting a new topic, you "forget" context from the previous topic.

**2. Input Gate** $i(t)$ and Candidate Cell $g(t)$ — decides what new information to add:
$$i(t) = \sigma(W_{xi}\, x(t) + W_{hi}\, h(t-1) + b_i)$$
$$g(t) = \tanh(W_{xg}\, x(t) + W_{hg}\, h(t-1) + b_g)$$

$g(t)$ is the candidate new content; $i(t)$ controls how much of it gets written in.

**3. Output Gate** $o(t)$ — decides what to read from long-term memory:
$$o(t) = \sigma(W_{xo}\, x(t) + W_{ho}\, h(t-1) + b_o)$$

#### State Update Equations

**Update the cell state** (erase old, write new):
$$c(t) = f(t) \odot c(t-1) + i(t) \odot g(t)$$

where $\odot$ denotes element-wise multiplication.

**Compute hidden state** (read from cell state):
$$h(t) = o(t) \odot \tanh(c(t))$$

**Output:**
$$y(t) = W_{hy}\, h(t) + b_y$$

#### Why LSTM Solves Vanishing Gradients

In a Vanilla RNN, the gradient flows back through many matrix multiplications involving $W_{hh}$ — causing exponential decay. In LSTM, the cell state update is **additive**:
$$c(t) = f(t) \odot c(t-1) + i(t) \odot g(t)$$

Gradients can flow backward through this addition relatively unchanged, as long as the forget gate is not near zero. The LSTM creates a **gradient highway** through time.

#### Weight Matrices Summary

| Weight | Connects | Description |
|--------|---------|-------------|
| $W_{xi}, W_{xf}, W_{xo}, W_{xg}$ | Input x(t) to gates/candidate | 4 input weight matrices |
| $W_{hi}, W_{hf}, W_{ho}, W_{hg}$ | Previous h(t-1) to gates/candidate | 4 recurrent weight matrices |
| $b_i, b_f, b_o, b_g$ | — | 4 bias vectors |

Total parameters: 4x more than a Vanilla RNN of the same hidden size.

---

### 4.5 GRU — Gated Recurrent Unit

Introduced by Cho et al. (2014). GRU simplifies LSTM by merging the cell state and hidden state into a single state $h(t)$, and reducing three gates to two.

#### Two Gates

**1. Reset Gate** $r(t)$ — controls how much of the past hidden state to use when computing the candidate:
$$r(t) = \sigma(W_{xr}\, x(t) + W_{hr}\, h(t-1) + b_r)$$

**2. Update Gate** $z(t)$ — controls the balance between retaining old state and adopting the new candidate:
$$z(t) = \sigma(W_{xz}\, x(t) + W_{hz}\, h(t-1) + b_z)$$

#### Candidate and State Update

**Candidate hidden state** (using reset gate to partially suppress the past):
$$\tilde{h}(t) = \tanh(W_x\, x(t) + W_h\,(r(t) \odot h(t-1)) + b)$$

**Updated hidden state** (interpolate between old and candidate):
$$h(t) = (1 - z(t)) \odot h(t-1) + z(t) \odot \tilde{h}(t)$$

*Intuition*: when $z(t) \approx 0$, the hidden state barely changes (past preserved); when $z(t) \approx 1$, the hidden state is fully replaced by the candidate (new information dominates).

---

### 4.6 LSTM vs GRU Comparison

| Aspect | Vanilla RNN | LSTM | GRU |
|--------|------------|------|-----|
| States | h(t) only | h(t) + c(t) | h(t) only |
| Gates | None | 3 (forget, input, output) | 2 (reset, update) |
| Parameters | Fewest | Most | Intermediate |
| Handles long-range dependencies | Poor | Excellent | Good |
| Training speed | Fastest | Slowest | Medium |
| Typical performance | Weakest | Best on complex tasks | Often matches LSTM |

**When to choose:**
- **Vanilla RNN**: short sequences, simple dependencies, very resource-constrained.
- **LSTM**: long sequences, complex long-range dependencies, when best performance is required.
- **GRU**: good balance of performance and efficiency; often matches LSTM with fewer parameters; preferred when computation is limited.

## 5. Cross-Session Concept Map

The three sessions form a coherent progression:

```
CORE IDEA (shared across all three):
  Learn a function f(x; theta) by minimising a loss via gradient descent + backpropagation

+-------------------+--------------------+---------------------+
|  SESSION I — MLP  | SESSION II — CNN   | SESSION III — RNN   |
+-------------------+--------------------+---------------------+
| Dense connections | Local connections  | Recurrent connect.  |
| Tabular/fixed data| Grid/spatial data  | Sequential/temporal |
| Fully learned feat| Shared filters     | Shared weights/time |
| Single forward    | Single forward     | Loop over time steps|
| Backprop          | Backprop           | Backprop Thru Time  |
| Any data type     | Images, 1D signals | Text, audio, video  |
+-------------------+--------------------+---------------------+
```

**Key conceptual threads running through all three:**

**Weight sharing**: CNN shares kernels spatially; RNN shares weights temporally. Both exploit regularity in the data structure to reduce parameters and improve generalisation.

**Inductive bias**: MLP has no structural assumptions (most flexible, least efficient). CNN assumes local spatial correlations. RNN assumes temporal order matters.

**Vanishing gradients**: A universal challenge in deep learning.
- MLP/CNN: Solved by ReLU activation + ResNet skip connections.
- RNN: Solved by LSTM/GRU gating mechanisms.

**Feature hierarchy**: MLP hidden layers learn progressively abstract features. CNN layers learn edges → textures → shapes → objects. RNN hidden states learn tokens → phrases → sentences → discourse.

**Architectural evolution timeline:**
```
1958: Perceptron (Rosenblatt)
1986: Backpropagation popularised (Rumelhart, Hinton, Williams)
1997: LSTM (Hochreiter & Schmidhuber)
1998: LeNet-5 (LeCun) — first practical CNN
2010: ReLU for deep networks (Nair & Hinton)
2012: AlexNet — deep learning revolution (Krizhevsky et al.)
2014: GoogLeNet / Inception Module (Szegedy et al.)
2015: ResNet / Skip connections (He et al.)
```

## 6. Key Equations Reference Sheet

### Neuron Output
$$y_k = \varphi\!\left(\sum_{j=0}^{m} \omega_{kj}\, x_j\right)$$

### Backpropagation Weight Update
$$\Delta\omega_{ji} = \eta\, \delta_j\, y_i$$

- **Output neuron**: $\delta_j = e_j \cdot \varphi'_j(v_j)$ where $e_j = d_j - y_j$
- **Hidden neuron**: $\delta_j = \varphi'_j(v_j) \sum_k \delta_k \omega_{kj}$

### Gradient Descent
$$\mathbf{x}' = \mathbf{x} - \epsilon\, \nabla f(\mathbf{x})$$

### Convolution (cross-correlation, 2D)
$$s(n,m) = \sum_{k,l} x(k,l)\, \omega(n+k,\, m+l)$$

### Vanilla RNN
$$h(t) = \tanh(W_{hh}\, h(t-1) + W_{xh}\, x(t) + b_h)$$

### LSTM
$$f(t) = \sigma(W_{xf} x(t) + W_{hf} h(t-1) + b_f)$$
$$i(t) = \sigma(W_{xi} x(t) + W_{hi} h(t-1) + b_i)$$
$$o(t) = \sigma(W_{xo} x(t) + W_{ho} h(t-1) + b_o)$$
$$g(t) = \tanh(W_{xg} x(t) + W_{hg} h(t-1) + b_g)$$
$$c(t) = f(t) \odot c(t-1) + i(t) \odot g(t)$$
$$h(t) = o(t) \odot \tanh(c(t))$$

### GRU
$$r(t) = \sigma(W_{xr} x(t) + W_{hr} h(t-1) + b_r)$$
$$z(t) = \sigma(W_{xz} x(t) + W_{hz} h(t-1) + b_z)$$
$$\tilde{h}(t) = \tanh(W_x x(t) + W_h (r(t) \odot h(t-1)) + b)$$
$$h(t) = (1 - z(t)) \odot h(t-1) + z(t) \odot \tilde{h}(t)$$

### Common Activation Functions

| Function | Formula | Derivative |
|----------|---------|------------|
| Sigmoid | $\frac{1}{1+e^{-v}}$ | $\varphi(v)(1-\varphi(v))$ |
| Tanh | $\tanh(v)$ | $1 - \varphi(v)^2$ |
| ReLU | $\max(0, v)$ | 1 if v > 0, else 0 |
| Softmax | $\frac{e^{v_k}}{\sum_j e^{v_j}}$ | $\varphi_k(\delta_{kj} - \varphi_j)$ |
