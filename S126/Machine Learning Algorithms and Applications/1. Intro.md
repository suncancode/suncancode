## 1. What is Machine Learning?

ML is the **automated process of extracting patterns from data** - programming computers to optimize a performance criterion using example data or past experience.

In general terms, ML algorithms learn a function that maps inputs from one space to outputs in another:
- A **learning algorithm** is one that can learn from data.
- It typically involves: an **optimization procedure**, a **cost function**, a **model**, and a **dataset**.
- There is a model defined up to some parameters; learning = optimizing those parameters using training data.
- Models can be **predictives** (make predictions on new data), **descriptive** (gain insight from data), or both.

### Mitchell's Definition of Learning (1997)
> A computer program is said to learn from **experience E** with respect to some class of **tasks T**, and **performance measure P**, if its performance at tasks in T as measured by P **improves with experience**.

Key points:
- Tasks of interest are "too difficult to solve with fixed programs written and designed by humans."
- Learning is **not** the task itself — it is the means of acquiring the ability to perform the task.

## 2. Common Learning Scenarios

*(Based on Mohri, Rostamizadeh & Talwalkar, 2012)*

| Scenario | Description |
|---|---|
| **Supervised Learning** | Learner receives labelled examples; makes predictions on unseen points. Used in classification, regression, ranking. |
| **Unsupervised Learning** | Learner receives only unlabelled data; learns structure. Examples: clustering, dimensionality reduction. |
| **Semi-supervised Learning** | Mix of labelled and unlabelled data. Used when labelled data is expensive but unlabelled data is abundant. |
| **Transductive Inference** | Like semi-supervised, but the goal is to label only the specific given test points, not all future inputs. |
| **Online Learning** | Training and testing are interleaved round-by-round. At each round: receive unlabelled point → predict → incur loss. Goal: minimize cumulative loss. |
| **Reinforcement Learning** | Learner interacts with an environment, receives rewards per action. Goal: maximize cumulative reward over time. Related to dynamic programming. |
| **Active Learning** | Learner adaptively queries an oracle to get labels for selected points. Goal: match supervised learning performance with fewer labels. |
| **Few-Shot Learning** | Emulates human ability to learn from very few examples. Methods range from adapting pre-trained models to meta-learning approaches. |

## 3. Classes of Learning Tasks

Tasks are described in terms of how the ML system should process examples. An **example** is collection of quantitatively measured features: **x ∈ ℝⁿ**.

### 3.1 Classification 
- Assign an input to one of *k* categories.
- Learn a function: **f : ℝⁿ → {1, …, k}**
- Output `y = f(x)` assigns feature vector `x` to a category.
- The function can also output a **probability distribution** over classes.

### 3.2 Regression
- Predict a numerical value given an input.
- Learn a function: **f : ℝⁿ → ℝ**
- Contrast with classification: output is continuous, not categorical.

### 3.3 Transcription
- Convert unstructured data (e.g., audio) into discrete text.
- Learn: **f : ℝⁿ → 𝔸^k(m)** where 𝔸 is a language alphabet set.
- Example: **speech recognition**.

### 3.4 Machine Translation
- Convert a sequence in source language to target language.
- Learn: **f : 𝔸ₓ^k(n) → 𝔸ᵧ^k(m)**

### 3.5 Structured Output
- Subsumes transcription and translation.
- Output is a data structure with inter-related values.
- Example: generating a text description of an image or video.

### 3.6 Anomaly Detection
- Flag unusual or atypical events from a dataset.
- Anomalous objects belong to a very different probability distribution.
- Key challenge: how to estimate distributions and measure distance between them.

### 3.7 Synthesis and Sampling
- Generate new examples similar to training data.
- Example: speech synthesis, image generation (GANs).
- *GAN example*: Given bedroom images as training data, a GAN can generate new realistic bedroom images; given text descriptions, it can synthesize matching bird images.

### 3.8 Imputation of Missing Values
- Predict missing entries `xᵢ` in an example vector **x ∈ ℝⁿ**.

### 3.9 Denoising
- Predict a clean example **x** from its corrupted version **x̃**.
- Equivalent to learning the conditional distribution: **p(x | x̃)**.

### 3.10 Density Estimation
- Learn a probability density (or mass) function:
  **p_model : ℝⁿ → ℝ**
- Underpins many other ML tasks.

## 4. Performance Measures

- Quantitative measures are required to evaluate ML algorithms.
- Performance is **task-specific**.
- Common examples: **Accuracy**, **Error Rate**, **Precision**, **Recall**.
- The performance measure must be chosen to **match the desired system behavior**.

## 5. Experience: Supervised vs Unsupervised

### Unsupervised Learning
- Receives dataset with many features; no labels.
- Goal: learn useful properties from the data.
- Formally: given examples of random vector **x**, learn the probability distribution **p(x)**.

### Supervised Learning
- Receives dataset with features **and** labels (targets) per example.
- Formally: given **x** and corresponding labels **y**, learn to estimate the conditional distribution **p(y | x)**.

### Discussion Points
1. How much data is needed to build a good ML system?
2. What happens when the distribution of training data differs from test data?
3. How can we quantify such a distribution mismatch?

## 6. Data Representation

Data is typically represented as a **design matrix** **X ∈ ℝ^{N×P}**:

$$\mathbf{X} = \begin{bmatrix} \mathbf{x}_1^T \\ \vdots \\ \mathbf{x}_N^T \end{bmatrix}$$

- **N** = number of examples (samples/observations)
- **P** = number of features per example
- Each example: **xᵢ = [x₁, …, x_P]^T**

When examples have variable numbers of features, a **set notation** is used:
$$\{\mathbf{x}^{(1)}, \mathbf{x}^{(2)}, \ldots, \mathbf{x}^{(m)}\}$$

## 7. Linear Regression Example

### The Task
Build a system that predicts a scalar **y ∈ ℝ** from a feature vector **x ∈ ℝ^P**.

**Model:** Output is a linear function of input:
$$\hat{y} = \boldsymbol{\omega}^T \mathbf{x}$$

More generally (with bias term):
$$\hat{y} = \boldsymbol{\omega}^T \mathbf{x} + b$$

where **ω ∈ ℝ^P** is the weight vector — each element determines how each feature affects the prediction.

### Training and Testing
- Split data into **training set** X^(train) and **test set** X^(test).
- Train on X^(train), evaluate generalization on X^(test).

**Performance measure — Mean Squared Error (MSE):**
$$\text{MSE}_\text{test} = \frac{1}{N_\text{test}} \sum_i \left(\hat{y}^{(\text{test})} - y^{(\text{test})}\right)_i^2$$

**Training objective:** Find **ω** minimizing MSE on training data:
$$\min_{\boldsymbol{\omega}} \frac{1}{P} \|\hat{\mathbf{y}}^{(\text{train})} - \mathbf{y}^{(\text{train})}\|_2^2 \quad \text{s.t.} \quad y = \boldsymbol{\omega}^T \mathbf{x}$$

**Closed-form solution** (differentiating and setting gradient to zero):
$$\boldsymbol{\omega} = \left(\mathbf{X}^{(\text{train})T} \mathbf{X}^{(\text{train})}\right)^{-1} \mathbf{X}^{(\text{train})T} \mathbf{y}^{(\text{train})}$$

This equation constitutes a simple learning algorithm.

## 8. Capacity, Overfitting and Underfitting

**Generalization** is the central goal: a trained model must perform well on **new, previously unseen inputs**.

| Concept | Definition |
|---|---|
| **Training error** | Error on data used during training |
| **Generalization error (test error)** | Expected error on new inputs drawn from the same data distribution |
| **Underfitting** | Model cannot achieve sufficiently low error on the *training* set |
| **Overfitting** | Gap between training error and test error is too large |
| **Capacity** | Ability of the model to fit a wide variety of functions |

### The i.i.d. Assumption
Training and test data are assumed to be **independent and identically distributed (i.i.d.)**, generated by the same underlying data-generating process.

### Controlling Overfitting and Underfitting
- **Alter capacity**: change the hypothesis space (e.g., polynomial degree, number of features).
  - Low capacity → underfitting
  - High capacity → overfitting
- **Increase training data**: more examples reduce overfitting.

### The Triple Trade-off *(Alpaydin, 2010)*
Three interacting quantities in any learning algorithm:
1. **Complexity** of the hypothesis / capacity of the hypothesis class
2. **Amount of training data** available
3. **Generalization error** on new examples

> *Example: Fitting a degree-9 polynomial to 4 data points will memorize the points but fail to generalize (overfitting). A linear fit may miss the true pattern (underfitting). A quadratic fit may generalize well.*

## 9. PAC Learning Framework

> *Note: This theoretical framework can be revisited after gaining deeper intuitive understanding.*

**PAC = Probably Approximately Correct**

### Setup and Definitions

| Symbol | Meaning |
|---|---|
| **𝒳** | Input space — set of all possible examples |
| **𝒴** | Output (label) space — e.g., {0, 1} for binary classification |
| **c : 𝒳 → 𝒴** | A concept — the true mapping we want to learn |
| **𝒞** | Concept class — the set of all target concepts |
| **ℋ** | Hypothesis set — the set of functions the learner considers |
| **𝒟** | An unknown fixed distribution over 𝒳 |
| **S** | A training sample of m i.i.d. examples drawn from 𝒟 |

### The Learning Problem
The learner receives a labelled sample **S = {x₁, …, xₘ}** and must select a hypothesis **h_S ∈ ℋ** with small **generalization error** with respect to the true concept **c**.

### Generalization Error (Risk)
$$R(h) = \Pr_{x \sim \mathcal{D}}[h(x) \neq c(x)] = \mathbb{E}_{x \sim \mathcal{D}}[\mathbf{1}_{h(x) \neq c(x)}]$$

Since **𝒟** and **c** are unknown, we use the **empirical error**:

### Empirical Error
$$\hat{R}(h) = \frac{1}{m} \sum_{i=1}^{m} \mathbf{1}_{h(x_i) \neq c(x_i)}$$

**Key property:** For a fixed hypothesis h, the expectation of the empirical error equals the true generalization error:
$$\mathbb{E}[\hat{R}(h)] = R(h)$$

### PAC-Learnability
A concept class **𝒞** is **PAC-learnable** if there exists an algorithm **𝒜** and a polynomial function *poly(·)* such that for any **ε > 0**, **δ > 0**, for any distribution **𝒟**, and for any target concept **c ∈ 𝒞**, for sample size **m ≥ poly(1/ε, 1/δ, n, size(c))**:

$$\Pr_{S \sim \mathcal{D}^m}[R(h_S) \leq \varepsilon] \geq 1 - \delta$$

**Interpretation:** With probability at least **(1 − δ)**, the learned hypothesis has generalization error at most **ε**.

The PAC framework provides theoretical bounds on:
- Required **training sample size** *m*
- Gap between training and true error
- **Complexity** of the hypothesis space ℋ
- **Confidence** in the learned result

## 10. Loss Functions

### Definition
Loss functions are objective functions that are **minimized** during training. They measure how well the model's predictions match the targets.

> *"Finding appropriate loss functions is one of the most important research endeavours in machine learning."* — Ciampiconi et al., 2023

### General Formulation
Goal: learn a function **f** mapping input space **Φ** to output space **𝒴**:
$$f : \Phi \mapsto \mathcal{Y}$$

The function is approximated by a **parameterized model** **f_Θ**.

### Per-sample Loss
A loss function **L** maps a prediction **f(x)ᵢ** and the true label **yᵢ** to a scalar **l ∈ ℝ** capturing their dissimilarity.

### Overall (Aggregate) Loss
$$\mathcal{L}(f \mid \{x_0, \ldots, x_N\}, \{y_0, \ldots, y_N\}) = \frac{1}{N+1} \sum_{i=0}^{N} L(f(x_i),\, y_i)$$

### Optimization Problem
$$\min_f \mathcal{L}(f \mid \{x_0, \ldots, x_N\}, \{y_0, \ldots, y_N\})$$

### Regularization
To prevent overfitting, a **regularization term** **R(f)** is added:
$$\min_f \frac{1}{N+1} \sum_{i=0}^{N} L(f(x_i), y_i) + \mathcal{R}(f)$$

In terms of parameters **Θ**:
$$\min_\Theta \frac{1}{N+1} \sum_{i=0}^{N} L(f_\Theta(x_i), y_i) + \mathcal{R}(\Theta)$$

We search the **parameter space** for values that minimize the regularized loss.
