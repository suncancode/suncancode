## 1. Pattern Recognition / Classifier

### 1.1 What is Pattern Recognition?

Pattern recognition studies how to design machines that can **recognize and classify "things"** in the world — despite variety in their form and existence. This includes:
- Studying the **statistics of features** that describe objects
- Measuring and selecting **good classification systems**

### 1.2 Basic Model of a Pattern Classifier

```
Sensor → Feature Selector/Extractor → Classifier → Decision
         (Representation Pattern)    (Feature Pattern)
```

- **Representation Pattern**: Raw data from the sensor (e.g., image pixels, stock prices)
- **Feature Pattern**: A small set of variables obtained via *feature selection and/or extraction*
- **Classifier**: Uses the feature pattern to make a decision

A **pattern** is a p-dimensional vector:

$$\mathbf{x} = [x_1 \quad x_2 \quad \cdots \quad x_p]^t$$

Examples of patterns:
- Pixels in an image
- Stock market closing prices
- Speech signal recordings
- Weather variable measurements
- Real estate measurements
- Behavioural/lifestyle data

### 1.3 Class Notation

Assume there are **C classes** denoted: $\omega_1, \ldots, \omega_C$

A variable $z$ indicates class membership:
> If $z = i$, then the pattern $x$ belongs to $\omega_i$, where $i \in \{1, \ldots, C\}$

### 1.4 Classifier Design

**Problem**: Given a training set of patterns of known class, design a classifier optimal for expected operating conditions.

Design entails:
- Specifying the **classifier model parameters**
- Ensuring **optimal response** for a given pattern

Key principles:
1. The training set is **finite**
2. The model must **not be too complex** → avoids over-fitting
3. Optimal performance on the **training set is NOT the goal**
4. **Generalization performance** is the primary goal

> **Key Assumption**: Training samples are drawn from the **same probability distribution** as test and operational samples.

### 1.5 Occam's Razor

> *Favour simpler explanations over those that are needlessly complicated.*

This principle underpins the popular **sparse representation** method.

## 2. Approaches to Classification

### 2.1 Supervised vs. Unsupervised Classification

| Type | Description |
|------|-------------|
| **Supervised** | Training data has labels (class type). Uses exemplars. |
| **Unsupervised** | Data is unlabelled. Goal: find groups and distinguishing features. |
| **Semi-supervised** | Uses both labelled and unlabelled data for training. |

### 2.2 Supervised Classification Example (Fish Sorting)

*From Duda, Hart & Stork (2001)*

**Task**: Automatically sort **salmon** from **bass** for a fishing company.

Possible features:
- Length, Width
- Number and shape of fins
- Position of mouth
- Lightness

**Single feature (length)**:
- Bass is generally longer than salmon
- Histogram of lengths shows overlap → decision threshold $L^*$ minimizes errors

**Single feature (lightness)**:
- Better separation than length
- Decision threshold $X^*$ gives fewer errors

**Two features (lightness + width)**:
- Feature vector: $\mathbf{x} = [x_1, x_2]^T$
- Feature space can be visualized
- Finding the **best decision boundary** is the classifier design problem

**Risks of complexity**:
- More features → **dimensionality curse** (too many features)
- Over-complex model → **over-fitting** (perfect on training, poor on new data)

## 3. Elementary Decision Theory

### 3.1 Bayes Decision Rule — Minimum Error

**Assumptions**: Full knowledge of the probability density function (PDF) of each class.

- Each class has a known **prior probability**: $P(\omega_1), \ldots, P(\omega_C)$
- Assign $x$ to class $\omega_j$ if the **posterior probability** $P(\omega_j | x)$ is highest

**Decision rule**:

$$x \in \omega_j \quad \text{if} \quad P(\omega_j | x) > P(\omega_k | x), \quad k = 1, \ldots, C; \; k \neq j$$

**Bayes' Theorem** expresses posterior in terms of prior and class-conditional density:

$$P(\omega_i | x) = \frac{p(x | \omega_i) P(\omega_i)}{p(x)}$$

where:
$$p(x) = \sum_{j=1}^{C} p(x | \omega_j) P(\omega_j)$$

**Decision rule in terms of class-conditional density** — assign $x$ to $\omega_j$ if:

$$p(x|\omega_j)P(\omega_j) > p(x|\omega_k)P(\omega_k), \quad k = 1, \ldots, C; \; k \neq j$$

**For a two-class case**, use the **likelihood ratio** $L_r(x)$:

$$L_r(x) = \frac{p(x|\omega_1)}{p(x|\omega_2)} > \frac{P(\omega_2)}{P(\omega_1)} \implies x \in \omega_1$$

### 3.2 Bayes Decision Rule — Minimum Risk

Define a **loss matrix** $\Lambda$ with components:

$$\lambda_{ji} = \text{cost of assigning pattern } x \text{ to } \omega_i \text{ when } x \in \omega_j$$

**Conditional risk** of assigning $x$ to class $\omega_i$:

$$l_i(x) = \sum_{j=1}^{C} \lambda_{ji} P(\omega_j | x)$$

**Average risk** over decision region $\Omega_i$:

$$r_i = \int_{\Omega_i} l_i(x) p(x) \, dx = \int_{\Omega_i} \sum_{j=1}^{C} \lambda_{ji} P(\omega_j|x) p(x) \, dx$$

**Overall risk**:

$$r = \sum_{i=1}^{C} r_i$$

**Minimum risk rule**: Assign $x \in \Omega_i$ if:

$$\sum_{j=1}^{C} \lambda_{ji} P(\omega_j|x) p(x) \leq \sum_{j=1}^{C} \lambda_{jk} P(\omega_j|x) p(x), \quad k = 1, \ldots, C$$

**Bayes risk** (minimum achievable):

$$r^* = \int_x \min_{i=1,\ldots,C} \sum_{j=1}^{C} \lambda_{ji} P(\omega_j|x) p(x) \, dx$$

**For two-class case**, minimum risk → decide $\omega_1$ if:

$$(\lambda_{11} - \lambda_{12}) p(x|\omega_1) P(\omega_1) < (\lambda_{22} - \lambda_{21}) p(x|\omega_2) P(\omega_2)$$

**Special case — Zero-one loss matrix**:

$$\lambda_{ij} = \begin{cases} 1, & i \neq j \\ 0, & i = j \end{cases}$$

Under this symmetric loss, minimum risk **reduces to** minimum error:

$$p(x|\omega_i) p(\omega_i) \geq p(x|\omega_k) p(\omega_k), \quad k = 1, \ldots, C$$

The conditional risk with zero-one loss:

$$l_i(x) = 1 - P(\omega_i|x)$$

### 3.3 Discriminant Functions

Bayes rules require knowledge of priors and class-conditional densities — often unavailable in practice.

**Alternative**: Assume a form of the **discriminant function** directly.

A discriminant function $h(x)$ leads to a classification rule:
$$h(x) > k \Rightarrow x \in \omega_1, \quad h(x) < k \Rightarrow x \in \omega_2$$

For **C classes**, define $C$ discriminant functions $g_i(x)$ such that:

$$g_i(x) > g_j(x) \Rightarrow x \in \omega_i, \quad j = 1, \ldots, C; \; j \neq i$$

> A feature vector is assigned to the class with the **largest discriminant**.

Key properties:
- Discriminant functions are **not unique** (any monotonic transformation preserves decisions)
- Rely on **function form**, not underlying distribution
- Parameters are set by a **training procedure**

### 3.4 Linear Discriminant Functions

A linear discriminant is a **linear combination** of feature vector components:

$$g(x) = \omega^t x + \omega_0 = \sum_{i=1}^{p} \omega_i x_i + \omega_0$$

- $\omega$: weight vector
- $\omega_0$: threshold weight (bias)
- Describes a **hyperplane** with unit normal $\omega$ and perpendicular distance $|\omega_0|/|\omega|$ from origin

The **signed distance** from the hyperplane is $g(x)/|\omega|$.

**Classifiers using linear discriminants** are called **linear machines**.

#### Minimum-Distance Classifier (Nearest-Neighbour)

Let prototype points be $p_1, \ldots, p_C$ (one per class).

Assign $x$ to $\omega_i$ with nearest prototype:

$$\|x - p_i\|^2 = x^t x - 2x^t p_i + p_i^t p_i$$

Equivalent linear discriminant:

$$g_i(x) = \omega_i^t x + \omega_{i0}, \quad \text{where} \quad \omega_i = p_i, \quad \omega_{i0} = -\frac{1}{2}\|p_i\|^2$$

> If prototype points are **class means** → **nearest class mean classifier**

**Properties**:
- Decision boundaries are **perpendicular bisectors** between prototype points
- Decision regions of a linear machine are always **convex**

### 3.5 Piecewise Linear Discriminant Functions

Linear machines **cannot** separate non-convex decision regions.

**Solution**: Allow **multiple prototypes per class** ($n_i$ prototypes for class $\omega_i$):

$$g_i(x) = \max_{j=1,\ldots,n_i} g_i^j(x)$$

where each subsidiary function is:

$$g_i^j(x) = x^t p_i^j - \frac{1}{2} p_i^{j^t}, \quad j = 1, \ldots, n_i; \; i = 1, \ldots, C$$

## 4. Support Vector Machine (SVM)

### 4.1 Kernel Methods

**Main idea**: Embed data into a **higher-dimensional feature space** where nonlinear patterns become linear.

**Two steps**:
1. Define a mapping via a **kernel function** (encodes domain knowledge)
2. Apply a **general-purpose, robust algorithm**

**Advantage**: Algorithm complexity is polynomial in data size; embedding space can grow exponentially without increasing computational cost.

A kernel function $\phi$ transforms: nonlinear data in input space → linearly separable in feature space.

### 4.2 Ridge Regression — Foundation for Kernelization

Optimization problem:

$$\min_W F(W) = \lambda\|W\|^2 + \|X^T W - Y\|^2$$

- $\lambda$: regularization parameter (trade-off between regularization and MSE)
- $X \in \mathbb{R}^{N \times m}$: matrix of feature vectors

**Primal solution** (global minimum):

$$W = (XX^T + \lambda I)^{-1} XY$$

**Dual form** (via Gram matrix $G = X^T X$):

$$W = X\alpha, \quad \alpha = (G + \lambda I)^{-1} Y$$

where each entry of $G$ is an **inner product**: $G_{i,j} = \langle x_i, x_j \rangle$

**Prediction**:

$$g(x) = Y^T (G + \lambda I)^{-1} k, \quad k_i = \langle x_i, x \rangle$$

| Form | System Size | When Preferred |
|------|-------------|----------------|
| Primal | $N \times N$ | $N \ll m$ |
| Dual | $m \times m$ | $N \gg m$ |

> **Key observation**: Ridge regression can be solved using only **inner products** between sample points.

### 4.3 Kernel-Defined Nonlinear Mapping

Consider embedding map $\phi: x \in \mathbb{R}^N \mapsto \phi(x) \in F \subseteq \mathbb{R}^N$

The Gram matrix in feature space:

$$G_{i,j} = \langle \phi(x_i), \phi(x_j) \rangle$$

**Definition — Kernel Function**:

> A **kernel** $\kappa$ satisfies: $\kappa(x, z) = \langle \phi(x), \phi(z) \rangle$ for all $x, z \in S$

> The explicit form of $\phi(x)$ need **not** be known — the kernel computes the inner product directly.

#### Commonly Used Kernels

| Kernel | Mathematical Form $\kappa(x_i, x_j)$ | Parameters |
|--------|---------------------------------------|------------|
| Linear | $\langle x_i, x_j \rangle$ | — |
| Polynomial | $(\gamma\langle x_i, x_j \rangle + r)^d$ | $\gamma > 0$ |
| RBF (Gaussian) | $\exp(-\gamma\|x_i - x_j\|^2)$ | $\gamma > 0$ |
| Sigmoid | $\tanh(\gamma\langle x_i, x_j \rangle + r)$ | — |

#### Example — RBF Kernel

Let $x = [1, 4, 6]^T$, $z = [3, 5, 2]^T$, $\gamma = 0.2222$:

$$\kappa(x, z) = \exp(-0.2222 \times \|x - z\|^2) = \exp(-4.667) = 9.4 \times 10^{-3}$$

**Kernalization** allows computing inner products in the feature space without explicitly mapping:

$$G_{i,j} = \langle \phi(x_i), \phi(x_j) \rangle = \kappa(x_i, x_j)$$

> Note: **Feature space is not uniquely determined** by the kernel function.

### 4.4 Support Vector Machine (SVM) — Hard Margin

**Binary classification** with labels $y_i = \pm 1$ and decision function:

$$g(x) = \text{sign}(\langle w, x \rangle + b)$$

For a **separable dataset**, correct classification requires: $y_i(\langle w, x_i \rangle + b) > 0 \; \forall i$

**Canonical hyperplane**:
- $\langle w, x \rangle + b = +1$ for closest positive-class points ($x_1$)
- $\langle w, x \rangle + b = -1$ for closest negative-class points ($x_2$)
- Separating hyperplane: $\langle w, x \rangle + b = 0$

**Margin**:

$$\gamma = \frac{1}{\|w\|}$$

**Optimization** (maximize margin = minimize $\|w\|$):

$$\min \frac{1}{2}\|w\|^2 \quad \text{subject to} \quad y_i(\langle w, x_i \rangle + b) \geq 1 \; \forall i$$

**Primal Lagrangian**:

$$\mathcal{L} = \frac{1}{2}\langle w, w \rangle - \sum_{i=1}^{m} \alpha_i \left(y_i(\langle w, x_i \rangle + b) - 1\right)$$

**Dual objective** (maximize w.r.t. $\alpha_i$):

$$\mathcal{W}(\alpha) = \sum_{i=1}^{m} \alpha_i - \frac{1}{2} \sum_{i,j=1}^{m} \alpha_i \alpha_j y_i y_j \langle x_i, x_j \rangle$$

Subject to:
$$\alpha_i \geq 0, \quad \sum_{i=1}^{m} \alpha_i y_i = 0$$

### 4.5 Kernelized SVM

Replace inner products with a kernel function to handle **non-separable / nonlinear** data:

$$\langle x_i, x_j \rangle \mapsto \kappa(x_i, x_j) = \langle \phi(x_i), \phi(x_j) \rangle$$

**Kernelized dual objective**:

$$\mathcal{W}(\alpha) = \sum_{i=1}^{m} \alpha_i - \frac{1}{2} \sum_{i,j=1}^{m} \alpha_i \alpha_j y_i y_j \kappa(x_i, x_j)$$

**Decision function for a new test point $z$**:

$$f(z) = \text{sign}\left(\sum_{i=1}^{m} y_i \alpha_i \kappa(x_i, z) + b\right)$$

### 4.6 Soft Margin SVM

**Problem**: Noisy data and outliers can reduce generalization.

**Solution**: Introduce a **soft margin** to allow some misclassification.

Two approaches:
1. **L1 norm error**: Introduce box constraint $0 \leq \alpha_i \leq C$
2. **L2 norm error**: Add small constant to kernel diagonal: $\kappa(x_i, x_j) + \lambda$

**C** and $\lambda$ control the **bias-variance trade-off** — tuned via a validation set.

### 4.7 Practical Guide to Using SVM

*(Based on Hsu, Chang & Lin, 2003–2016)*

1. **Transform** data to SVM package format
2. **Scale** the data (simple normalization)
3. Start with the **RBF kernel**: $\kappa(x_i, x_j) = \exp(-\gamma\|x_i - x_j\|^2)$
4. Use **cross-validation** to find the best $C$ and $\gamma$
5. **Train** on the full training set with best parameters
6. **Test** on held-out data
