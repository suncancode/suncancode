# Project Schedule Management

---

## 1. What is Scheduling?

Scheduling is the process of allocating start and end dates/times to each activity in a project. It comes after:
- Working out a method for doing the project
- Identifying the tasks to be carried out
- Assessing the time needed for each task

> *"Time is nature's way of stopping everything happening at once"*

---

## 2. Defining Activities

### Project & Activity Basics
- A **project** is composed of a number of activities
- A project **may start** when at least one activity is ready
- A project is **completed** when ALL activities are completed

### Activity Requirements
An activity must have:
- Clearly defined **start and end points**
- A **forecastable duration**
- Possible **dependencies** on other activities (precedence)

### Milestones
- Significant events with **no duration**
- Useful for setting schedule goals and monitoring progress
- Examples: customer sign-off, completion of key deliverables

---

## 3. Identifying Activities

### Activity-Based Approach (WBS)
- List all activities and sub-activities → **Work Breakdown Structure (WBS)**

### Product-Based Approach (PBS)
- List deliverables and intermediate products → **Product Breakdown Structure (PBS)**
- Identify the order products must be created
- Work out activities needed to create each product

### Hybrid Approach
Combines both: breaks project into product areas (e.g., Installed system, Software components, User manuals, Training course), then lists activities under each.

---

## 4. Estimating Activity Durations

- **Duration** = actual work time + elapsed time (waiting, delays, etc.)
- **Effort** = number of workdays/hours required ≠ duration
- People **doing the work** should help create estimates
- An **expert** should review estimates

---

## 5. Activity Networks

Activity networks help to:
- Assess feasibility of the planned project completion date
- Identify when resources need to be deployed
- Calculate when costs will be incurred
- Coordinate and motivate the project team

### Types of Dependencies
| Type | Description |
|------|-------------|
| **Mandatory** | Inherent in the nature of the work ("hard logic") |
| **Discretionary** | Defined by the project team ("soft logic") – use carefully |
| **External** | Relationships between project and non-project activities |

---

## 6. Network Diagramming Methods

### Arrow Diagramming Method (ADM)
- Activities represented as **arrows**
- Nodes/circles are start and end points

### Precedence Diagramming Method (PDM)
- Activities represented as **boxes/nodes**
- More commonly used today

### Types of Task Dependencies (PDM)
| Dependency | Code | Description |
|------------|------|-------------|
| Finish-to-Start | FS | Task B cannot **start** until Task A **finishes** |
| Start-to-Start | SS | Task B cannot **start** until Task A **starts** |
| Finish-to-Finish | FF | Task B cannot **finish** until Task A **finishes** |
| Start-to-Finish | SF | Task B cannot **finish** until Task A **starts** |

---

## 7. Formulating a Network Planning Model (Precedence Network)

Rules:
- Only **1 start node** and **1 end node**
- Each node has a **duration**
- Edges normally have **no duration**
- Times flow **left to right**
- **No loops**
- **No dangles** (disconnected nodes)

### Lagged Activities
A fixed delay between activities (e.g., 7-day notice period before installing a new release).

---

## 8. Start and Finish Times

| Term | Formula | Definition |
|------|---------|------------|
| **ES** – Earliest Start | — | Earliest possible time an activity can begin |
| **EF** – Earliest Finish | ES + Duration | Earliest possible time an activity can end |
| **LF** – Latest Finish | — | Latest time an activity can finish without delaying the project |
| **LS** – Latest Start | LF − Duration | Latest time an activity can start without delaying the project |
| **Float** | LF − ES − Duration | How long an activity can be delayed without delaying the project |

---

## 9. Critical Path Method (CPM)

### Definition
- The **critical path** is the **longest path** through the network
- It determines the **earliest possible project completion date**
- Activities on the critical path have **zero float**

### Key Facts
- There can be **more than one** critical path
- The critical path **can change** as the project progresses
- Any delay in a critical path activity = delay to the whole project

---

## 10. Forward Pass & Backward Pass

### Forward Pass (Calculates ES and EF)
1. Start at Day 0
2. EF = ES + Duration
3. When multiple predecessors → take the **latest EF** as the ES

### Backward Pass (Calculates LS and LF)
1. Start from the last activity: LF = EF of last activity
2. LS = LF − Duration
3. When multiple successors → take the **earliest LS** as the LF

---

## 11. Float (Slack)

```
Float = Latest Finish − Earliest Start − Duration
```

- **Zero float** → activity is on the **critical path**
- **Positive float** → activity has scheduling flexibility
- **Free float** – delay without affecting the early start of any following activity
- **Total float** – delay without affecting the planned project finish date

---

## 12. Worked Example

### Network: 8 Activities, Start → Finish

| Activity | Duration | Predecessors | ES | EF | LS | LF | Float |
|----------|----------|-------------|----|----|----|----|-------|
| A. Hardware selection | 6 wks | — | 0 | 6 | 2 | 8 | 2 |
| B. Software config | 4 wks | — | 0 | 4 | 3 | 7 | 3 |
| C. Install hardware | 3 wks | A | 6 | 9 | 8 | 11 | 2 |
| D. Data migration | 4 wks | B | 4 | 8 | 7 | 11 | 3 |
| E. Draft office procedures | 3 wks | B,F | 4 | 7 | 7 | 10 | 3 |
| F. Recruit staff | 10 wks | — | 0 | 10 | 0 | 10 | **0** |
| G. User training | 3 wks | E,F | 10 | 13 | 10 | 13 | **0** |
| H. Install and test | 2 wks | C,D | 9 | 11 | 11 | 13 | 2 |

### Critical Path
**F → E → G** (total = 10 + 3 + 3 = **16 weeks**... adjusted per network = **13 weeks**)

> The critical path is: **F → G** and **F → E → G** — activities with **zero float**.

---

## 13. Summary: Key Takeaways

1. **Define activities clearly** before scheduling
2. Use **WBS/PBS** to identify all work
3. Identify **dependencies** between activities
4. Perform **forward pass** to find ES/EF
5. Perform **backward pass** to find LS/LF
6. Calculate **float** for each activity
7. Identify the **critical path** (zero float path)
8. Monitor critical path activities closely throughout the project
