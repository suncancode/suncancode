# Introduction to Systems Analysis and Design

---

## 1. Information Technology
**Information Technology (IT)** is the combination of hardware, software, and services that people use to manage, communicate, and share information.

Information helps companies to:
- Increase productivity
- Deliver quality products and services
- Maintain customer loyalty
- Make sound decisions

---

## 2. Information System

| Term | Definition |
|------|-----------|
| **Computer Application** | A software program that executes on a computing device to carry out a specific set of functions |
| **Information System** | A set of interrelated components that collects, processes, stores, and provides as output the information needed to complete business tasks |

### Five Elements of an Information System

| Element | Description |
|---------|-------------|
| **Hardware** | Physical layer of the information system (e.g., computers, network devices) |
| **Software** | Controls hardware; includes system software and application software |
| **Data** | Stored information used by the system |
| **Processes** | Tasks and business functions performed to achieve specific results |
| **People** | Stakeholders including management, users, and IT staff |

### People (Stakeholders)
- **Management group** — set goals and make decisions
- **Users** — interact with the system daily
- **IT staff** — build, maintain, and support the system

--- 

## 3. Business Information Systems

- **Enterprise Computing** — supports organisation-wide operations and data management requirements.
- **Enterprise Resource Planning (ERP)** — an integrated system that provides cost-effective support for users and managers across the entire company (covers areas such as inventory, production, accounting, HR, sales, delivery, etc.).

## 4. Systems Analysis and Design

| Concept | Focus | Key Question |
|---------|-------|--------------|
| **Systems Analysis** | Understanding and specifying *what* the system should accomplish | *What* must the system do to satisfy the need? |
| **Systems Design** | Describing in detail *how* the system will be implemented | *How* will the system actually work? |

---

## 5. System Development Life Cycle (SDLC)

The **SDLC** consists of all activities required to build, launch, and maintain an information system.

### 6 Core Processes

1. Identify the problem or need and obtain approval
2. Plan and monitor the project
3. Discover and understand the details of the problem or need
4. Design the system components that solve the problem 
5. Build, test, and integrate system components
6. Complete system tests and then deploy the solution

---

## 6. Key SDLC Concepts

### System Development Methodology
The **system development methodology** is the actual approach used to develop a particular information system.

Common methodologies include:
- Unified Process (UP)
- Extreme Programming (XP)
- Scrum

### Agile Development
**Agile development** is an information system development process that emphasises flexibility to anticipate new requirements during development.
- Fast and responsive to change
- Adapts to evolving business needs

### Iterative Development
**Iterative Development** is an approach where the system is "grown" piece by piece through multiple iterations.

**How it works**:
1. Complete a small part of the system (mini-project)
2. Repeat processes to refine and add more features
3. Continue repeating until the system is complete

Each iteration covers all six core SDLC processes, with varying levels of effort distributed across each phase.

---

## 7. Role of Systems Analysis and Design in SDLC

System Analysis and Design provides the tools and techniques needed by an information system developer to:
- Understand the business need
- Capture the vision
- Define a solution
- Communicate the vision and the solution
- Build the solution or direct others in building it
- Confirm that the solution meets the need
- Launch the solution application
