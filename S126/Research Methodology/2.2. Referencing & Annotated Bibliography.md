# Referencing & Annotated Bibliography

---

## 1. Referencing — Why It Matters

- When you use information from others, you **must cite it** and provide the source information
- **Reference lists** are located at the **end of the work** and provide full source details
- Failure to cite is considered **plagiarism**

### Common Referencing Styles

| Style | Commonly Used In |
|-------|-----------------|
| **Harvard** | General academia |
| **IEEE** | Computer Science / Engineering |
| **Springer** | Springer publications |
| **APA** | Social sciences |
| **Chicago** | Humanities |

---

## 2. Harvard Referencing

### General Format
```
Last name, First Initial Year, Title, Edition, Publisher, City.
```
- Citations are listed in **alphabetical order** by the author's last name
- Multiple sources by the same author are ordered by **date of publication**

---

### Books

#### One Author — First Edition
```
Patterson, J 2005, Maximum ride, Little, Brown, New York.
```

#### One Author — Not First Edition
```
Dahl, R 2004, Charlie and the chocolate factory, 6th edn, Knopf, New York.
```

#### Multiple Authors
```
Desikan, S & Ramesh, G 2006, Software testing, Dorling Kindersley, Bangalore, India.

Daniels, K, Patterson, G & Dunston, Y 2014, The ultimate student teaching guide,
2nd edn, SAGE Publications, Los Angeles.
```

#### Edited Book
```
Susilo, W & Mu, Y (eds.) 2014, The 19th Australasian Conference on Information
Security and Privacy (ACISP 2014), Lecture Notes in Computer Science 8544,
Springer Verlag, Berlin.
```

---

### Chapters in Edited Books

**Format:**
```
Chapter Author, Initial(s) Year, 'Chapter Title', in Editor Initial(s) Surname
(ed./eds), Book Title, Publisher, Place, page numbers.
```

**Example:**
```
Baek, J, Susilo, W & Zhou, J 2008, 'Fuzzy Identity-based Encryption: New and
Efficient Schemes', in Y Li, S Ling, H Niederreiter, H Wang, C Xing & S Zhang
(eds), Series on Coding Theory and Cryptology - Vol. 4, World Scientific
Publishing, Singapore, pp. 1-14.
```

---

### Journal Articles

**Format:**
```
Author, Initial(s) Year, 'Article Title', Journal Title, volume number,
issue number, page numbers.
```

**Example:**
```
Guo, F, Susilo, W & Mu, Y 2004, 'Generalized Closest Substring Encryption',
Designs, Codes and Cryptography, 80(1), pp. 103-124.
```

---

### Journal Articles Found Online / Database

**Format:**
```
Author, Initial(s) Year, 'Article Title', Journal Title, volume/issue,
page numbers, viewed Day Month Year, <URL>.
```

**Example:**
```
Raina, S 2015, 'Establishing Correlation Between Genetics and Nonresponse',
Journal of Postgraduate Medicine, Volume 61(2), p. 148, accessed 8 April 2005,
available at: http://www.proquest.com/
```

---

### Conference Proceedings

**If not published online:**
```
Last name, First initial Year, 'Title of Paper', In: Name of Conference,
City, Publisher, pages.
```

**If published online:**
```
Last name, First initial Year, 'Title of Paper', In: Name of Conference.
[online] City, Publisher, pages, available at: URL [Accessed Day Mo. Year].
```

**Examples:**
```
Jiang, Y, Susilo, W, Mu, Y & Guo, F 2016, 'Ciphertext-Policy Attribute Based
Encryption Supporting Access Policy Update', In: The 10th International
Conference on Provable Security (ProvSec 2016), Nanjing, China, Lecture Notes
in Computer Science, Springer, pp. 39-60.

Palmer, L, Gover, E & Doublet, K 2013, 'Advocating for Your Tech Program',
In: National Conference for Technology Teachers. [online] New York, NCTT,
pp. 33-34, available at: http://www.nctt.com/ [Accessed 11 Jan. 2014].
```

---

### eBooks and PDFs
```
Author, Initial(s) Year, Title, Publisher, [ebook/pdf], available at: URL
[Accessed Day Month Year].
```

**Example:**
```
Zusack, M 2015, The Book Thief, 1st ed. [ebook] Knopf, New York,
available at: http://ebooks.nypl.org/ [Accessed 20 Apr. 2015].
```

## 3. In-text Citations

### Rules
- Only use the **first listed author's name** followed by **"et al."** when there are more than two authors
- Cite an article if the content **is related to or originally comes from** that source
- The cited article **must appear** in the reference list

### Citation Styles

#### Style 1 — Parenthetical / Harvard
Place author name and year in parentheses at the end of the sentence:

```
"Knowledge of the stages of growth and development helps predict the patient's
response to illness" (Potter et al., 2013).
```

#### Style 2 — Narrative
Integrate the author's name into the sentence:

```
Potter et al. (2013) go on to explain that "among the most Catholic Filipinos,
parents keep the newborn inside the home until after the baptism."
```

#### Style 3 — Index Number (IEEE Style)
Use a numbered reference placed inline:

```
The proposed scheme achieves better performance than previous approaches [12].
```

> Place the reference number in the **proper position** in the related sentence.

## 4. Annotated Bibliography

### Definition
An annotated bibliography is a bibliography that presents a **summary (and sometimes evaluation)** of each source.

### Purpose
- Provides a **concise summary** of each source
- Makes it **easy to reference** sources during writing
- **Essential for scientific writing** — helps track what you have read and why it matters

---

## 5. Types of Annotations

There are three main types:

```
Annotations
├── 1. Summary Annotations
│   ├── Informative
│   └── Indicative
├── 2. Evaluative Annotations
└── 3. Combination Annotations  ← most common
```

---

### ① Summary Annotations

Provide a summary of the source content — **do not evaluate**.

#### Informative Annotations
- Provide a straight, complete **summary** of the source material
- Summarise author's main points and relevant information
- Writing structure:
  ```
  Thesis statement → Argument/Hypothesis → Proofs/Evidence → Conclusion
  ```

#### Indicative Annotations
- Do **not** provide actual data or information from the source
- Only describe the **kinds of questions or issues** addressed (e.g., through chapter titles)
- No attempt to give hypotheses, proofs, or specific results

---

### ② Evaluative Annotations

Go beyond summarising — provide **critical appraisal**:

- Assess the source's **strengths and weaknesses** in terms of usefulness and quality
- Evaluate for **biases, lack of evidence**, or unclear objectives
- Show how the work **may or may not be useful** for a particular field or audience
- Explain how the material **assisted your own project**

---

### ③ Combination Annotations ← Most Common

Combines summary + evaluation + analysis:

- **Summarise or describe** the topic
- **Evaluate** the source's usefulness
- Include a detailed analysis of the **reason the article was written**

> Most annotated bibliographies in research use this type.

---

## 6. Format of an Annotated Bibliography

### Structure

```
[Bibliographic information — written in the chosen referencing style]

    [Annotation — written in paragraph form, 100–200 words]
    - Summary of key content
    - Evaluation of strengths and weaknesses (if evaluative)
    - Relevance to your research project
```

### Guidelines
- Bibliographic information uses a **hanging indent**
- Annotation length: **100 to 200 words** typically
- When writing an extensive analysis, more space may be needed
- A useful formula: *a few sentences of general summary + several sentences on how it fits your larger project*

### Sample Annotated Bibliography Entry

```
Hacker, A 2012, 'Is Algebra Necessary?', New York Times, July 31.

    This article, published in the New York Times, argues whether algebra is
    necessary in terms of hacking. The author presents both sides of the debate,
    examining scenarios where algebra may or may not be essential. The conclusion
    provides a balanced view of the pros and cons of algebra in this context.
    This source is useful for establishing the foundational debate around the
    relevance of mathematical skills in computer science education.
```

## 7. LaTeX for Academic Writing

### What is LaTeX?
- Based on **TeX** — an editing tool using **plain text with markup tags**
- Defines document structure, styled text, citations, and cross-references via commands
- **Widely used in academia**, especially for:
  - Mathematical formulas and equations
  - Structured scientific documents
  - Thesis and journal submissions

### LaTeX Setup

| Component | Options |
|-----------|---------|
| **Engine** | TexLive, MikTeX, MacTeX (for Mac) |
| **Editor** | TexStudio, Texmaker, WinEdt, Textpad, TeXShop, LaTeXiT |
| **Other tools** | PDF reader, Ghostview (PostScript viewer) |

---

### File Structure

LaTeX requires **two files** for bibliography:

```
project/
├── mybib.bib     ← bibliography file (references + annotations)
└── main.tex      ← main document file
```

---

### The `.bib` File — Bibliography with Annotations

```bibtex
@Misc{hacker,
  AUTHOR       = {Andrew Hacker},
  TITLE        = {Is Algebra Necessary?},
  HOWPUBLISHED = {New York Times},
  MONTH        = {July 31},
  YEAR         = {2012},
  ANNOTATE     = {This article was published in the New York Times.
                  The contents argue whether algebra is necessary at all
                  in terms of hacking. The conclusion provides pros and
                  cons of with and without algebra.},
}
```

---

### The `.tex` File — Main Document

#### With Annotated Bibliography
```latex
\documentclass[12pt]{article}
\title{Sample Annotated Bibliography}
\begin{document}
  \nocite{*}                        % print all bibliography entries
  \bibliographystyle{annotate}      % uses annotate.bst package
  \bibliography{mybib}
\end{document}
```

#### Without Annotations (Standard Bibliography Only)
```latex
\documentclass[12pt]{article}
\title{Sample Bibliography}
\begin{document}
  \nocite{*}
  \bibliographystyle{abbrv}         % standard abbreviated style
  \bibliography{mybib}
\end{document}
```

> Change `annotate` to `abbrv` (or `plain`, `ieeetr`, etc.) to switch bibliography style.

## 8. EndNote for MS Word

### What is EndNote?
- Referencing software designed for use with **MS Word**
- Allows you to create your own **reference library**
- Automatically **inserts citations** and **generates reference lists**
- Available at: http://endnote.com (30-day free trial)

### Workflow

```
1. Install EndNote
         ↓
2. MS Word displays the EndNote tab/icon
         ↓
3. Open EndNote → Build your library
   (upload from database OR enter manually)
         ↓
4. Open MS Word → write your document
         ↓
5. Insert citation → load reference from EndNote
         ↓
6. EndNote auto-generates the reference list
```

---

### EndNote Library Format

```
%0 Journal Article
%A Herbert H. Clark
%D 1982
%T Hearers and Speech Acts
%B Language
%V 58
%P 332-373

%0 Book
%A Geoffrey Chaucer
%D 1957
%T The Works of Geoffrey Chaucer
%I Houghton
%C Boston
%N 2nd

%0 Thesis
%A Cantucci, Elena
%T Permian strata in South-East Asia
%D 1990
%I University of California, Berkeley
%9 Dissertation
```

### Common EndNote Field Tags

| Tag | Field Name |
|-----|-----------|
| `%0` | Reference type |
| `%A` | Author |
| `%B` | Secondary title / Conference name |
| `%C` | Place published |
| `%D` | Year |
| `%E` | Editor / Secondary author |
| `%I` | Publisher |
| `%J` | Journal name |
| `%V` | Volume |
| `%P` | Pages |

### Common Reference Types (`%0`)

| Code | Type |
|------|------|
| `%0 Book` | Book |
| `%0 Journal Article` | Journal article |
| `%0 Book Section` | Chapter in a book |
| `%0 Thesis` | Thesis / Dissertation |
| `%0 Blog` | Blog post |
| `%0 Bill` | Legal bill |
| `%0 Artwork` | Artwork |
| `%0 Government Document` | Government document |

```
    ├── Reference management for MS Word
    ├── Build a library → insert citations → auto-generate reference list
    └── Supports all major reference types
```

### Golden Rules
1. **Always cite** — using others' information without citation is plagiarism
2. Use a **consistent referencing style** throughout your document
3. Start building your **bibliography from day one** using BibTeX or EndNote
4. Annotated bibliography length: **100–200 words per entry**
5. Most annotations should be **combination type** — summarise AND evaluate
6. In Harvard style, use **"et al."** for more than two authors in in-text citations
7. **LaTeX** is the recommended tool for thesis and journal writing in CS/IT
