1. What is XSLT?

XSLT (Extensible Stylesheet Language Transformations) is a language based on XML, designed to transform XML documents into other formats, such as HTML, plain text, or even another XML structure. It plays a crucial role in web development by enabling dynamic presentation of data stored in XML files. XSLT files typically use the .xsl extension and are processed by an XSLT processor to generate the desired output.

Key features of XSLT include:

  • Flexible transformation of XML data into various formats.
  • Support for conditional logic and looping to manipulate data.
  • Integration with XPath for navigating and selecting XML elements.

2. Basic Structure of an XSLT File

An XSLT stylesheet begins with an XML declaration followed by the <xsl:stylesheet> element. This element defines the transformation rules and includes essential attributes: 

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0"
    xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
    xmlns="http://www.w3.org/1999/xhtml">
    <!-- Transformation rules go here -->
</xsl:stylesheet>

- version="1.0": Specifies the XSLT version. - xmlns:xsl: Declares the XSLT namespace. - xmlns: Ensures the output conforms to XHTML standards.

3. How XSLT Interacts with XML

XSLT interacts with XML by applying transformation rules to convert the hierarchical structure of an XML document into a different format. This process involves:

  1. Input XML Document: The source XML file contains the raw data (e.g., exam timetable).
  2. XSLT Stylesheet: Defines how to map XML elements to the target format (e.g., HTML).
  3. XSLT Processor: A tool (e.g., browser or server-side engine) that executes the transformation.
  4. Output: The resulting file (e.g., HTML) ready for display.

Example: Transforming an Exam Timetable

XML Input:

<exam subject="MATH 2113">
    <title>Abstract Algebra</title>
    <venue>20.G01</venue>
    <date>15/11/2018</date>
    <time>1PM-4PM</time>
    <note>Closed book exam, calculator allowed</note>
</exam>

XSLT Stylesheet:

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform" xmlns="http://www.w3.org/1999/xhtml">
    <xsl:template match="/">
        <html>
            <body>
                <h2>Exam Information</h2>
                <p>Subject: <xsl:value-of select="/exam/@subject"/></p>
                <p>Title: <xsl:value-of select="/exam/title"/></p>
                <p>Venue: <xsl:value-of select="/exam/venue"/></p>
                <p>Date: <xsl:value-of select="/exam/date"/></p>
                <p>Time: <xsl:value-of select="/exam/time"/></p>
                <p>Note: <xsl:value-of select="/exam/note"/></p>
            </body>
        </html>
    </xsl:template>
</xsl:stylesheet>

HTML Output:

<html>
    <body>
        <h2>Exam Information</h2>
        <p>Subject: MATH 2113</p>
        <p>Title: Abstract Algebra</p>
        <p>Venue: 20.G01</p>
        <p>Date: 15/11/2018</p>
        <p>Time: 1PM-4PM</p>
        <p>Note: Closed book exam, calculator allowed</p>
    </body>
</html>

In this example, the <xsl:template> element matches the root of the XML document, and <xsl:value-of> extracts the values of XML elements and attributes to build the HTML output.
