Lecture 11B - Form Validation

1. Objectives

  • Understand and implement built-in form validation using HTML5 attributes.
  • Use JavaScript to perform custom form validation.

2. Built-in Form Validation

HTML5 provides built-in validation using attributes to enforce data integrity without JavaScript.

  • required: Ensures a field must be filled out.
  • min and max: Sets minimum and maximum values for numbers.
  • minlength and maxlength: Sets minimum and maximum length for text.
  • pattern: Defines a regular expression for input validation (e.g., email format).
  • type="email", type="number": Validates input type automatically.
<input type="email" required pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$" />

3. JavaScript Form Validation

JavaScript allows custom validation logic for more complex requirements.

  • checkValidity(): Checks if all input elements in a form are valid.
  • setCustomValidity(message): Sets a custom error message.
  • Events: onsubmit, onchange, oninput for real-time validation.
document.getElementById("myForm").onsubmit = function() {
  if (!this.checkValidity()) {
    alert("Please fill out the form correctly!");
    return false;
  }
};

4. Practical Applications

Form validation ensures data quality in web applications, such as:

  • Registration forms with email and password checks.
  • Order forms with quantity and price validation.
  • Real-time feedback for user input.
