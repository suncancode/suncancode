Web Storage Lecture

1. Introduction to Web Storage

Web Storage is an HTML5 API for storing data in the browser without server interaction. It replaces cookies with a larger capacity (typically 5-10MB).

  • localStorage: Data persists until manually cleared.
  • sessionStorage: Data lasts only for the current tab session.

2. Comparison of localStorage and sessionStorage

  • localStorage: Permanent storage, accessible across tabs/windows of the same domain.
  • sessionStorage: Session-specific, cleared when the tab closes, not shared between tabs.
  • Both store data as key-value pairs and support only strings.

3. Basic Methods

  • setItem(key, value): Stores a key-value pair.
  • getItem(key): Retrieves the value for a key.
  • removeItem(key): Removes a specific key-value pair.
  • clear(): Removes all stored data.
  • length: Returns the number of stored items.
localStorage.setItem("username", "John");
console.log(localStorage.getItem("username")); // Output: John
localStorage.removeItem("username");

4. Practical Applications

Web Storage is used for:

  • Saving user preferences (e.g., dark mode).
  • Storing temporary form data or shopping cart items.
  • Synchronizing data across pages.

5. Limitations and Security

  • Only stores strings; use JSON.stringify and JSON.parse for complex data.
  • Limited storage (5-10MB per domain).
  • Data is not encrypted; accessible via JavaScript, requiring security considerations.
