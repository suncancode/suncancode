Introduction to Forms

HTML forms are used to collect user input and contain various form elements like text fields, checkboxes, etc. When submitted, data is sent to the server for processing. Front-end elements must match back-end parameters.

Example: Whosvilleâ€™s Library Book Search

Back-end at http://library.whosville/bsearch with parameters: author, year, sub (multiple values: mth, cs, bio, phy, chem).

Submitted URL example: http://library.whosville/bsearch?author=tonien&year=2000&sub=cs&sub=phy.

<form action="http://library.whosville/bsearch" method="get">
    <input type="checkbox" name="sub" value="mth"/> Mathematics<br>
    <input type="checkbox" name="sub" value="cs"/> Computer Science<br>
    <!-- other checkboxes -->
    <input type="text" name="author"/><br>
    <input type="text" name="year"/><br>
    <input type="submit" value="Search"/>
</form>

Note: Use name for parameter names, value for data sent to server. Displayed text can differ from sent value.

Form Elements

Type Text

<input type="text" name="firstname" size="30"/>

Type Password

<input type="password" name="password" size="30"/>

Type Checkbox

<input type="checkbox" name="journal" value="AMM"/> American Mathematical Monthly

Use checked="checked" for pre-selected.

Type Radio

<input type="radio" name="studentType" value="u"/> Undergraduate

Same name for group. Use checked="checked" for default.

Select (Dropdown)

<select name="day">
    <option value="mon">Monday</option>
    <!-- options -->
</select>

Use selected="selected" for default.

Select Multiple

<select name="day" multiple>
    <option value="mon">Monday</option>
    <!-- options -->
</select>

Textarea

<textarea name="comment" rows="5" cols="30"></textarea>

Submit and Reset

<input type="submit" value="Login"/>
<input type="reset" value="Reset form"/>

Reset clears all form inputs.

Form Methods: GET vs POST

AspectGETPOST
VisibilityData visible in URLData not displayed
HistoryParameters in browser historyNot saved
BookmarkedCan be bookmarkedCannot
SecurityLess secure (data in URL)More secure
Back/ReloadHarmlessMay resubmit (alert)
Data LengthLimited (2048 chars)No limit
Data TypeASCII onlyBinary allowed

Use POST for sensitive data like passwords.
