JavaScript Basics

This website introduces the fundamental of JavaScript, a programming language used to make web pages interactive. It covers basic syntax, inclusiojn methods, variables, datatypes, operators, etc.

1. My First JavaScript

JavaScript can be triggered by events like button clicks.
Here is an example:

        <button onClick="sayHi();">
        Click me
        </button>

        </script>
        function sayHi() {
            alert("hi");
            console.log("hi");
            console.log(1+2);
        }
        </script>

Explanation and Usage: The onClick attribute calls the sayHi() function when the button is clicked. alert() shows a pop-up message, and console.log() prints to the browser console (viewable in developer tools, F12 in most browsers). Use this to test simple interactions or debug code.

2. Where to Include JavaScript

JavaScript code can be placed anywhere in HTML, but common practices are in the <head> or at the end of the <body>.

  • In the Head: For code that needs to load early.
    •         <head>
        <title>JavaScript Example</title>
        <script>
            function sayHi(){
                alert("Hi");
            }
        </script>
        </head>
  • At the End of Body: For better page loading performance.
    •         <script>
            function sayHi(){
                alert("Hi");
            }
        </script>
        </body>
        </html>

Explanation and Usage: Placing scripts at the end of <body> prevents blocking HTML rendering. Use <head> for scripts that define functions needed early. Always wrap code in <script> tags.

3. External JavaScript

Instead of embedding code, link to an external .js file:

        <script type="text/javascript" src="js/myscript.js"></script>

(In myscript.js:)

        function sayHi(){
            alert("Hi");
        }

Explanation and Usage: This separates code from HTML for reusability. Place the .js file in a folder like "js/" and link it in your HTML. Ideal for larger projects to keep HTML clean.

4. Basic JavaScript Syntax

Statements end with semicolons (;). Comments use // or /* */.

        /* This is a multi-line comment */
        function silly(){
            alert('Hi'); // Display alert
            console.log(2+2); // Print 4
        }

Explanation and Usage: Semicolons separate statements. Comments document code without execution. Use // for single lines and /* */ for blocks. Always comment complex code for clarity.

5. Variables

Use var to declare variables (dynamic typing).

        var studentName = "John";
        var x, y;
        x = 5;
        y = x + 2;

Explanation and Usage: Variables store data. JavaScript is case-sensitive (studentName â‰  StudentName). Dynamic typing means variables can change types (e.g., number to string). Use camelCase or underscore for naming. Undeclared variables are undefined.

6. Data Types: Number

Numbers can be integers or floats.

        var age = 19;
        var pi = 3.14;

Arithmetic operators: + (addition), - (subtraction), * (multiplication), / (division), % (modulus).

        console.log(5 + 3); // 8
        console.log(10 % 3); // 1

Explanation and Usage: Use for calculations. % gives remainder (useful for even/odd checks). Numbers are dynamic, so no type declaration needed.

7. Data Types: String

Strings use single or double quotes.

        var age = "19";
        var name = 'John';

Explanation and Usage: Strings are text. Use + for concatenation (e.g., "Hello " + name). Quotes must match (single or double).

Final Notes: Practice in browser console or tools like JSFiddle. JavaScript runs client-side, making pages interactive.
