JavaScript Events

1. Events Overview

Events control user interactions (e.g., clicks, mouse movements). Associate handlers with HTML elements using attributes like onClick, onMouseOver.

        <button onClick="handlerFunction1()">Click me</button>
        <span onMouseOver="handlerFunction2()">some text</span>

Explanation and Usage: When an event occurs (e.g., click), the specified function runs. Use functions for complex logic. Attach to buttons, images, or text for interactivity. Visual Demo: Hover over this text: Hover me!

2. onClick Event

Triggered when an element (button, image) is clicked.

        <script>
            function cat(){
                alert("Don’t click me, click the dog!");
            }
            function dog(){
                alert("I don’t mind being clicked!");
            }
        </script>

        <img onClick="cat()" src="cat.png" />
        <img onClick="dog()" src="dog.png" />

Explanation and Usage: Useful for buttons or links. Attach to for clickable images. Visual Demo:

3. onLoad Event

Triggered after an element's resource loads (e.g., page or image).

        <script>
            function greeting(){
                alert("Welcome to my website!");
            }
        </script>

        <body onLoad="greeting()">

Explanation and Usage: Use on for page load actions (e.g., welcome message). On for image load confirmation. Ideal for initializing scripts after content loads.

4. onFocus Event

Triggered when an element gains focus (e.g., clicking into a text field).

        <input type="text" onFocus="..." />
        <textarea onFocus="..."></textarea>
        <select onFocus="..."></select>

Explanation and Usage: Use for form validation or hints (e.g., change background on focus). Attach to inputs for user feedback. Visual Demo:

5. onBlur Event

Triggered when an element loses focus (e.g., tabbing out of a field).

        <input type="text" onBlur="..." />
        <textarea onBlur="..."></textarea>
        <select onBlur="..."></select>

Explanation and Usage: Use for validation after input (e.g., check email format on blur). Pairs well with onFocus for form interactions. Visual Demo:
