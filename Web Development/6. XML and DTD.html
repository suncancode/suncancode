Objectives

  • Use XML to store and transport data over the Internet.
  • Learn DTD language to define the structure of an XML document.

Introduction to XML

XML (EXtensible Markup Language) is a markup language similar to HTML, used for storing and transporting data. It is software- and hardware-independent, separates data from presentation, and uses the .xml file extension.

Difference from HTML: HTML has predefined tags, while XML allows user-defined tags. Standards are defined using DTD or XSD.

Comparison with JSON: JSON is another popular data interchange format. Example:

{
  "firstName": "John",
  "lastName": "Smith",
  "email": "jsmith@gmail.com",
  "mobile": "0211223344"
}

In web apps, XML/JSON store/transport data, while HTML/XSLT transform/display it.

XML Structure and Rules

XML Declaration

Optional but recommended, must be the first line (no blank lines before). Syntax:

<?xml version="1.0" encoding="UTF-8" standalone="yes" ?>

Example:

<?xml version="1.0" ?>
<student>
  <firstName>John</firstName>
  <lastName>Smith</lastName>
  <email>jsmith@gmail.com</email>
  <mobile>0211223344</mobile>
</student>

Root Element

Every XML document must have one root element as the parent of all others.

Not Well-Formed (No Root):

<?xml version="1.0" encoding="UTF-8"?>
<student>...</student>
<student>...</student>

Well-Formed (With Root):

<?xml version="1.0" encoding="UTF-8"?>
<studentList>
  <student>...</student>
  <student>...</student>
</studentList>

Elements and Attributes

An element includes start and end tags, can have attributes. Example:

<?xml version="1.0" encoding="UTF-8"?>
<dailyTransaction date="24/02/2015">
  <person staffDbId="103" operation="update">
    <firstName>John</firstName>
    <lastName>Smith</lastName>
    <mobile>0211223344</mobile>
  </person>
</dailyTransaction>

Attributes must be quoted (single or double quotes).

Case Sensitivity: <student> differs from <STUDENT>.

Attribute vs Child Element

Metadata (e.g., gender) as attributes, data as elements. Example:

<person gender="M">
  <firstName>John</firstName>
  <lastName>Smith</lastName>
</person>

Better as:

<person>
  <firstName>John</firstName>
  <lastName>Smith</lastName>
  <gender>M</gender>
</person>

Empty Elements and Nesting

Empty elements use self-closing tags: <emptyElement />. All elements must be properly nested.

Entity References

Use to replace special characters:

CharacterEntityMeaning
<&lt;Less than
>&gt;Greater than
&&amp;Ampersand
'&apos;Apostrophe
"&quot;Quotation mark

Comments

Use: <!-- this is a comment -->

Introduction to DTD

DTD defines the legal structure of an XML document, can be internal or external.

Internal DTD Example

<?xml version="1.0" standalone="yes" ?>
<!DOCTYPE student [
  <!ELEMENT student (firstName,lastName,email,mobile)>
  <!ELEMENT firstName (#PCDATA)>
  <!ELEMENT lastName (#PCDATA)>
  <!ELEMENT email (#PCDATA)>
  <!ELEMENT mobile (#PCDATA)>
]>
<student>
  <firstName>John</firstName>
  <lastName>Smith</lastName>
  <email>jsmith@gmail.com</email>
  <mobile>0211223344</mobile>
</student>
