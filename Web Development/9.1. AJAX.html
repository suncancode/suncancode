Lecture 9A - AJAX

1. Introduction to AJAX

AJAX (Asynchronous JavaScript and XML) is a set of web development techniques using various web technologies on the client-side to create asynchronous web applications. It allows web pages to be updated asynchronously by exchanging data with a web server behind the scenes, avoiding full page reloads.

Scenario Example: Building a website about Wollongong to display dynamic information on Accommodation, Attractions, Events, Restaurants, Timetable, and Weather without reloading the page.

2. How AJAX Works

AJAX uses the XMLHttpRequest object (or modern Fetch API) to send HTTP requests to the server asynchronously. The server responds with data (often JSON or XML), which JavaScript processes and updates the DOM.

  • Asynchronous: Requests don't block the UI.
  • Technologies Involved: JavaScript, XMLHttpRequest/Fetch, DOM, JSON/XML.
// Basic AJAX flow with Fetch API
fetch('data.json')
  .then(response => response.json())
  .then(data => {
    // Update DOM with data
    document.getElementById('output').innerHTML = JSON.stringify(data);
  })
  .catch(error => console.error('Error:', error));

3. Using XMLHttpRequest (Traditional Method)

XMLHttpRequest is the original way to implement AJAX. It supports GET/POST requests and handles states (0-4: UNSENT, OPENED, HEADERS_RECEIVED, LOADING, DONE). 

var xhr = new XMLHttpRequest();
xhr.open('GET', 'https://api.example.com/weather', true); // true for async
xhr.onreadystatechange = function() {
  if (xhr.readyState === 4 && xhr.status === 200) {
    var data = JSON.parse(xhr.responseText);
    console.log(data);
  }
};
xhr.send();

5. Handling Errors and Best Practices

Always handle errors with .catch() or try-catch. Use JSON for data interchange (links to Lecture 8). 

// Error handling example
fetch('data.json')
  .then(response => {
    if (!response.ok) throw new Error('Network response was not ok');
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => alert('Error: ' + error));
  • Best Practices: Use async/await for readability, validate data, handle CORS issues.
